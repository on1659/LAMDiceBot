# 지시서 — 회전 칼날(spin-arena) 탈출 리프레이밍

결정서: `docs/dev-cycle/2026-06-11-spin-arena-win-condition-review.md` (먼저 읽기)

## 개요

탈락(HP 0)을 "죽음"에서 **"링 밖으로 튕겨나가 탈출 성공"** 으로 재해석한다. 끝까지 탈출 못 한 최후 1인 = 당첨(벌칙). **서버 `simulate()` 함수와 공유 상수는 코드 1줄도 변경 금지** — 같은 시드 = 같은 frames/eliminations/result가 회귀 증명이다. 당첨자는 변경 전후 **동일 인물**이어야 한다(기존 rank 1 = 새 최하위).

## 절대 불변조건

1. `socket/spin-arena.js`의 `simulate()` / `ringRadiusAt()` / 상단 공유 상수 블록: **코드 무변경** (주석 문구 갱신만 허용).
2. selected(당첨자) = 기존과 동일 인물. DB 기록(`isWinner = !isSelected`, rank 1/2, `recordServerGame`/`recordGameSession`) 시맨틱 무변경. `gameRules: 'spin-survival'` 문자열 유지.
3. 클라 `Math.random` 실호출 2회(deviceId/tabId) 유지. 신규 연출은 전부 t·페이로드 파생(2탭 동일).
4. 소켓 이벤트 이름/페이로드 구조 무변경 (rankings 배열의 정렬 순서·rank 번호 의미만 바뀜 — 클라는 name 조회라 안전).
5. 모바일/PC 양쪽: 캔버스 논리 480×480 유지, 패널 텍스트 길이 컴팩트.

## 1. 서버 — socket/spin-arena.js

### 1-1. rankHumans (258~274행) — 정렬 의미 플립
새 의미: **rank 1 = 가장 먼저 탈출(가장 먼저 탈락)**, 최하위 = 끝까지 못 나감 = 당첨.

기존 비교자의 **모든 절을 역전한 완전 역순**이어야 한다 (slotId tie-break 포함):
```javascript
// 사람 슬롯만 탈출순 랭킹(기존 생존순의 완전 역순 — selected 동일 인물 보장).
// 먼저 탈출(탈락)한 쪽 위 > 못 나간 쪽은 HP 낮은 순(탈출 임박) > slotId 높은 쪽 위(기존 tie-break의 역전)
humans.sort((a, b) => {
    if (a.alive !== b.alive) return a.alive ? 1 : -1;                   // 탈출(사망)이 위
    if (a.alive && a.hp !== b.hp) return a.hp - b.hp;                   // 둘 다 못 나감: HP 낮은 쪽 위
    if (!a.alive && a.elimMs !== b.elimMs) return a.elimMs - b.elimMs;  // 먼저 탈출한 쪽 위
    return b.slotId - a.slotId;                                         // 동률: slotId 역전 — 아래 주의 참조
});
```

**주의 — HP 동률은 0 반환 금지(fall-through 필수)**: `if (a.alive) return a.hp - b.hp;`처럼 쓰면 HP 동률에서 0이 반환돼 slotId 절에 도달하지 못하고, V8 안정 정렬이 입력순을 유지해 동률 엣지에서 당첨자가 legacy와 달라진다 (구 코드는 안정 정렬 의존으로 slotId 낮은 쪽이 [0]이었음 — 합성 테스트로 검증됨).
**왜 slotId까지 역전하나**: 기존 selected = 기존정렬[0], 동률이면 slotId 낮은 쪽이었다. 새 selected = 새정렬[last]이므로, 동률 그룹에서 slotId 낮은 쪽이 [last]에 가게 하려면 tie-break도 역전해야 한다. `a.slotId - b.slotId`로 두면 **동률 엣지(같은 HP 생존 2인 / 같은 elimMs)에서 당첨자 인물이 바뀐다.** 코드 주석에 이 사유를 남길 것.

## 2. 클라 — js/spin-arena.js

### 2-1. 탈락 트리거 분기 (912~926행)
reveal 페이로드의 `payload.result.selected`로 분기:
- **selected가 아닌 슬롯 (일반 = 탈출 성공)**: 축하 연출 — 기존 파편/링/플래시 재사용하되 텍스트 `'OUT'`(빨강) → `'탈출!'`(긍정색, 예 `#ffd24a` 골드 또는 `#7ddf8f`), 흔들림 약화(9 → 6), 사운드 `spin-arena_eliminate` 유지.
- **selected 슬롯 (동시 전멸 엣지 — 최후 2인이 같은 프레임 탈락)**: 실패 연출 — 텍스트 `'못 나감!'`(빨강 `#ff6b6b`), 기존 빨간 톤 파편 유지, 흔들림 유지(9). **이 분기가 없으면 당첨자에게 탈출 축하가 나가는 모순 발생** (Codex 지적 — 필수).

### 2-2. 장외 퇴장 애니메이션 (1053~1068행 시체 렌더 교체)
- 일반 탈출자: 탈락 후 회색 시체 영구 표시 대신 **상승+페이드 퇴장** — `t ∈ [elimMs, elimMs+700]` 동안 yOffset = 진행도×(−60px), alpha 1→0, 이후 미표시. 전부 t 파생(결정론, 2탭 동일). 스프라이트는 본인 스킨색 변형 유지(회색 금지 — 탈출은 성공이다).
- selected가 탈락한 엣지: 기존 회색 시체 페이드 유지(못 나가고 쓰러짐 — 실패 서사).

### 2-3. 게이지 인버트 — "탈출 게이지"
데이터는 동일(HP), **표시만 인버트**: 게이지 채움 = `(100 − hp) / 100`, 가득 차면 탈출.
- 캔버스 캐릭터 머리 위 바: 채움 = 100−hp, 색은 긍정 톤(스킨색 또는 골드 계열 — 기존 색 변수 재사용 우선). 칩 잔상(`chipHp`, 908~910행): 기존 로직 유지하되 표시 전환 — 솔리드 채움 = `100−chipHp`, 하이라이트 구간 = `[100−chipHp, 100−hp]`(방금 차오른 분량, 밝은 색)로 그린다. chipHp 갱신 코드는 무변경.
- HP 패널(1383~1430행): row fill도 동일 인버트. 탈출자 row: `'✓ ' + rank + '위 탈출'` (rank = 새 의미 = 탈출 순서), 회색 처리 유지. 진행 중 수치 표시는 % 또는 기존 형식 유지 중 단순한 쪽 — 의미만 "찰수록 탈출 임박"이면 됨.
- 패널 제목/레이블에 'HP'가 있으면 '탈출 게이지'로.

### 2-4. 막판 글로우 (1123~1132행)
