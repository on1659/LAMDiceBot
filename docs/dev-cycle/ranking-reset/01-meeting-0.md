# 팀 회의 결과: ranking-reset (사이클 #0)

## 팀원별 의견

### 지민 (PD · 10년차 senior)
**Go. 조건부.** 사설 서버 호스트의 시즌 리셋은 1순위 "기존 게임 UX 개선"에 해당. 프론트 1파일 + 백엔드 2파일 + 소켓 1곳. v1으로 1일 이내 구현, 배포 리스크 낮음. game_sessions는 v1에서 무시 가능, v2 백로그로. 첫 배포 후 24시간 모니터링 권고.

### 현우 (기획자① · 5년차 mid)
랭킹 고착화 문제 해결. 사용자 스토리: "사설 서버 호스트로서, 누적 랭킹을 초기화하고 싶다. 참여자들이 새 시즌처럼 경쟁 동기를 갖게." MoSCoW: Must(호스트만+2단계확인+실시간동기화), Should(알림메시지+캐시무효화), Could(초기화이력+시즌표시), Won't(부분초기화+자유플레이). 2단계 확인에서 삭제 건수 표시 제안.

### 소연 (기획자② · 8년차 senior)
**ICE 480점** (I:8 × C:10 × E:6). 검토 대상이나 범위가 작아 즉시 진행 가능. 호스트 리텐션 + 서버 활성도에 기여. v2로 시즌 아카이브, 초기화 로깅 제안. isHost 전달 경로(A: init 파라미터 vs B: setHost setter) 결정 필요.

### 태준 (백엔드 · 6년차 mid)
**수정 파일**: `db/ranking.js`, `routes/server.js`, `ranking-shared.js`. DELETE API + 호스트 검증(host_name 비교, 기존 패턴 재활용) + 소켓 broadcast. 트랜잭션으로 server_game_records 삭제. game_sessions도 함께 삭제 추천. 라우트 패턴 `/:serverId(\\d+)`로 자유 플레이 자연 차단. 공수 1.3일.

### 미래 (프론트 · 3년차 junior)
`ranking-shared.js`에 `setHost(isHost)` setter + 헤더에 🗑️ 버튼 + 인라인 확인바 추가. 4개 HTML 파일에서 setHost 호출부 추가 필요. z-index 충돌 우려로 모달보다 인라인 확인바 추천. 공수 약 1시간.

### 윤서 (QA · 5년차 mid)
**CRITICAL**: 권한 우회(API 직접 호출), 게임 진행 중 초기화 시 race condition. **HIGH**: game_sessions 누락, 배지 캐시(room.userBadges) 무효화 범위. Tier 1~3 검증 체크리스트 27항목 도출. game_sessions 처리 방침 + 게임 중 초기화 차단 여부 + broadcast 범위 확정 필요.

### 다은 (UI · 4년차 mid)
헤더 우측 🗑️ 버튼(38x38px, 터치 44x44px). 기존 .rk-back-btn과 동일 패턴(rgba(255,255,255,0.2)). 인라인 확인바: rgba(220,53,69,0.15) 배경 + [취소](회색) + [초기화](#dc3545 빨강). 성공/실패 피드백 2초 표시. 새 CSS 변수 불필요 — 기존 다크 테마 인라인 패턴 유지.

### 승호 (UX · 7년차 mid)
호스트/참여자 플로우 분리 설계. 🗑️ 탭 → 확인 다이얼로그 → 초기화 → 빈 상태 + 토스트. 참여자: 소켓으로 "호스트가 랭킹을 초기화했습니다" 토스트. 빈 랭킹 상태 "아직 기록이 없습니다" 안내 필수. aria-label, 키보드 접근성 체크.

## 주요 합의점

1. **Go 판정** — 전원 동의. 범위 작고 공수 낮음 (BE 1.3일 + FE 1시간)
2. **2단계 확인 필수** — 8명 전원 동의. 인라인 확인바 방식 선호 (모달 z-index 충돌 회피)
3. **서버사이드 호스트 검증 필수** — API 직접 호출 우회 방지. host_name 비교 패턴 재활용
4. **자유 플레이 제외** — 라우트 패턴 `/:serverId(\\d+)`로 자연 차단
5. **캐시 무효화** — 소켓 `rankingReset` broadcast + 클라이언트 `invalidateCache()`
6. **삭제 범위**: `server_game_records`만 (order_stats, vehicle_stats 유지)
7. **isHost 전달**: `setHost(isHost)` setter 방식 (init 시그니처 변경 최소화)

## 주요 충돌 지점

| 쟁점 | 의견 A | 의견 B | 결정 |
|------|--------|--------|------|
| **game_sessions 포함** | 함께 삭제 (태준, 윤서) — 고아 레코드 방지 | v1 무시 (지민) — game_sessions 직접 참조 기능 없음 | **사장 결정 필요** |
| **버튼 위치** | 헤더 우측 (다은, 미래) | 하단 (승호) — 오조작 방지 | **헤더 우측** (사장이 이미 결정) |
| **게임 진행 중 초기화** | 차단 (윤서) — race condition 방지 | 허용 (태준) — 게임 종료 후 새 INSERT는 정상 | **허용** (단순화 우선) |
| **삭제 건수 사전 표시** | 필요 (현우) — COUNT 후 다이얼로그에 표시 | 불필요 — API 추가 없이 단순 확인 | **불필요** (단순화) |
