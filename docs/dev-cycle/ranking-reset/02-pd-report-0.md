## 개발 범위 확정

### 이번 사이클 포함 (v1)
- 랭킹 오버레이 헤더 우측 🗑️ 버튼 (호스트만 표시)
- 2단계 인라인 확인바 ("랭킹을 초기화할까요?" → [취소] / [초기화])
- DELETE /api/ranking/:serverId API (호스트 검증 + server_game_records 삭제)
- 소켓 `rankingReset` broadcast → 클라이언트 캐시 무효화 + 리렌더
- 호스트에게 "랭킹이 초기화되었습니다" 피드백
- 참여자에게 "호스트가 랭킹을 초기화했습니다" 토스트 (소켓 수신 시)
- 자유 플레이(serverId=null) 버튼 미표시

### 제외 (v2 또는 미정)
- game_sessions 동기 삭제 (현재 직접 참조 없으므로 안전)
- 초기화 이력 로깅 (누가 언제 초기화했는지)
- 시즌 아카이브 (삭제 전 스냅샷 보관)
- 게임별 부분 초기화 (주사위만, 경마만 등)
- 삭제 건수 사전 표시

## 예상 사용 시나리오

### 개발 전 (As-Is)
사설 서버 호스트가 랭킹을 리셋하고 싶을 때, 방법이 전혀 없다. DB에 직접 접근할 수도 없고, 서버를 삭제하고 다시 만드는 것이 유일한 우회 방법이지만 서버 설정과 멤버까지 모두 잃게 된다.

### 개발 후 (To-Be)
호스트가 랭킹 오버레이를 열고 → 헤더 우측 🗑️ 버튼 탭 → "랭킹을 초기화할까요?" 확인바에서 [초기화] 탭 → 즉시 게임 기록 삭제. 주문통계/탈것통계는 유지. 같은 서버 접속자 전원에게 실시간 알림. 서버 설정·멤버·주문 기록은 그대로.

### 기대 효과
- 호스트 관리 편의성 향상: 시즌 리셋 0클릭 → 2클릭
- 참여자 경쟁 동기 재생성: 고착화된 랭킹 → 새 시즌 시작
- 서버 재생성 없이 리셋 가능 → 멤버/설정 보존
