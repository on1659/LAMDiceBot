## 1. 전제 검증 — 피드백의 사실관계

| 피드백 주장 | 판정 | 근거 |
|---|---|---|
| "일부러 빨리 죽는 전략이 최적해" | **성립 불가** | spin-arena는 플레이어 입력이 0인 결정론 관전형. 서버가 시드 PRNG로 30초를 사전 시뮬(`socket/spin-arena.js` `simulate()`), 클라는 리플레이만. 입력 이벤트는 스킨 선택/시작뿐 → 전략 자체가 존재하지 않음 |
| "최후 생존으로 벌칙 당첨 = 억울하다" | **실재하는 문제** | "오래 버팀 = 잘함"이라는 서사를 화면이 강화(생존 글로우, '최후 생존' 라벨)하면서 그 사람에게 벌칙을 줌 — 감정적 프레이밍 역전. Codex 보강 근거: DB는 이미 `isWinner = !isSelected`(socket/spin-arena.js:521)로 "최후 생존자 = 패자" 의미론을 기록 중 — **화면 서사만 거꾸로 말하고 있음** |

→ 핵심 문제는 "역인센티브"(기각)가 아니라 **"프레이밍 역전"**(채택)으로 재정의한다.

## 2. 안별 판정

| 항목 | 판정 | 근거 |
|---|---|---|
| 핵심 문제 제기 (High) | **변형 채택** | 위 1절 — 전략 문제가 아닌 프레이밍 문제로 재정의하여 해소 |
| 안 1: 탈출봉 5개 수집 (High) | **변형 채택(프레이밍만) + 아이템 메커니즘 보류** | "탈출 조건 달성" 방향은 채택하되, 아이템 경제는 보류. 사유: ① 입력이 없어 아이템 수집도 똑같이 운빨 — agency가 추가되지 않고 시각만 바뀜 ② 칼날이 킬을 못 하게 되면 킬 성장(+1, 상한 5) 메커니즘 사망 ③ 200시드 배치로 튜닝한 "30초 내 정확히 1명" 분포 자산 폐기 + 전면 재튜닝 ④ bridge-cross 교훈("메커니즘 통째 갈아엎지 말 것") |
| 안 2: 아이템 강탈 + 수집 (Mid) | **기각** | 명세대로면 수학적 데드락: 총량 3n 보존 시 n=6에서 3명 탈출 후 잔여 3개/3명 → 5개 영구 불가. 스폰을 추가하면 사실상 안 1로 수렴 — 독립 안으로 무의미 |
| 안 3: 물리 충돌 이동 (Low) | **기각(중복) + 일부 보류** | 초기 방향/속도 시드 랜덤(`simulate()` 초기화), 벽 반사(`WALL_BOUNCE` 0.9), 넉백 반사 모두 **이미 구현됨**. 유일하게 없는 캐릭터끼리 충돌 반사는 보류(밸런스 재튜닝 게이트를 다시 여는 비용 대비 효과 불확실) |

## 3. 채택안 — "탈출 리프레이밍" (시뮬 무변경)

**같은 시뮬레이션, 반대 서사.** 탈락(HP 0)을 "죽음"이 아니라 **"링 밖으로 튕겨나가 탈출 성공"** 으로 재해석한다.

- 새 룰 안내: "검에 맞아 기력이 다하면 링 밖으로 튕겨나가 **탈출 성공**! 끝까지 탈출 못 하고 링에 갇힌 **최후 1인이 당첨(벌칙)**."
- 당첨자는 지금과 **동일 인물**(최후 생존자) — 서버 시뮬·시드·밸런스·결정론·DB 승패 기록 전부 무변경.
- 바뀌는 것: 순위 의미(1위 = 첫 탈출), 탈락 연출(죽음 → 탈출 축하 + 장외 퇴장), 게이지 의미(체력 → 탈출 게이지), 막판 글로우(승자 금색 → 갇힘 위험 적색), 결과 라벨('통과/최후 생존' → '탈출 성공/탈출 실패'), 튜토리얼/메타 문구.
- 긴장 구조 보존: 탈출이 하나씩 터지는 긍정 이벤트 연쇄 → 마지막 2인에서 "둘 중 누가 못 나가나"로 클라이맥스 — 현 시뮬의 결판 연출(줌/무적 서사 보호)과 그대로 맞물림.

### 기각한 대안: "최초 탈락 = 당첨" 역전
첫 탈락(보통 10~15초)에 결판 → 남은 시간이 무의미한 안티클라이맥스. Codex도 동일 사유로 기각.

## 4. Codex 교차 검토 반영 사항

1. **동시 전멸 엣지의 연출 모순 (PARTIAL → 반영)**: 최후 2인 동시 탈락 엣지(클라 1674행 분기가 실존 증명)에서 당첨자에게도 '탈출 축하' 연출이 나가면 억울함이 부활 → **당첨자(selected) 슬롯은 탈출 연출 대신 'KO/못 나감' 실패 연출로 분기** (클라 전용, 시뮬 무변경).
2. **게이지 의미 인버트 (반영)**: HP바+칩 잔상은 "피해 = 나쁨" 격투 문법 → **탈출 게이지(차오르면 탈출)로 표시 인버트**. 데이터는 동일(표시 = 100−HP), 칩 잔상은 "방금 차오른 분량" 하이라이트로 전환.
3. **킬 플래시('⚔️+1') 톤 (인지, 최소 대응)**: 칼날 성장 축하가 "남을 탈출시켜주는 자해 신호"로 읽힐 수 있음 — 단 이 아이러니는 기존 프레이밍에도 있었고, 플래시는 칼날 재배치 스냅을 가리는 기능적 연출이라 **유지하되 정보성 톤 유지**(추가 축하 강화 금지).
4. **안 1 재평가 트리거 (반영)**: 리프레이밍 적용 후 다음 플레이테스트에서 **동일한 억울함 피드백이 재발하면 안 1(아이템 수집)을 High로 격상**해 재검토한다. 피드백 작성자에게는 "전략적 역인센티브는 입력 없는 구조상 성립하지 않으며, 억울함(프레이밍)을 탈출 구조로 해소했다"고 회신 권장.

## 5. 구현 범위 (지시서: `docs/dev-cycle/2026-06-11-spin-arena-escape-reframe-instructions.md`)

## 6. 불변조건 (구현에서 절대 유지)

- `simulate()`/공유 상수/시드/frames/eliminations **무변경** — 동일 시드 = 동일 결과.
- selected = 동일 인물(오늘의 rank 1 = 내일의 최하위). DB `isWinner`/rank/`recordServerGame` 시맨틱 무변경. `gameRules: 'spin-survival'` 유지(내부 메타데이터, 소비처 없음 확인).
- 클라 Math.random 실호출 2회(deviceId/tabId) 유지. 모든 신규 연출은 t·페이로드 파생(2탭 동일).
- reveal 전 server-only(timeline/result/seed) 비노출 — 이번 변경은 reveal 이후 표현만.
