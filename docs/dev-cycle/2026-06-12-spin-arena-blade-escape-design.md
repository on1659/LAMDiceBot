## 0. 한 줄 결론

**"받은 데미지 누적 T마다 칼 +1, 칼 5개 도달 = 즉시 탈출, HP 0 = 비석 + 3초 다운 후 부활(진행도 유지), 마지막 1명이 남는 순간(decideMs) + 2초 결판 비트에서 종료"** — 이동/칼날/넉백/링/판정 물리는 전부 무변경(정체성 보존), 바뀌는 것은 ① 칼 성장 원천(킬→데미지 임계) ② 죽음의 결과(영구 탈락→3초 다운) ③ 종료 조건(고정 30초→결판 즉시 + 30초 캡) 3가지다.

## 2. 대안 비교

### A안 (채택) — 받은 데미지 임계 수집 + 부활 + 종료 압축
- 받은 데미지 누적 `BLADE_UP_DMG`(T)마다 칼 +1. 칼 `ESCAPE_BLADES`(5)개 = 즉시 탈출.
- 플레이버: **"검에 맞으면 상대 검기를 흡수해 내 검이 된다"** — 기존 "맞을수록 게이지가 참" 시각 문법과 완전 정합.
- 당첨자 의미론 연속: 현행(가장 덜 맞고 버틴 사람 = selected)과 동일 인물 유형 — 직전 리프레이밍·DB 시맨틱과 충돌 없음.
- 효과: Effort M / Risk Mid / 기존 물리·밸런스 자산 최대 재활용.

### B안 (기각) — 가한 데미지 수집
- "검 = 무기 = 때려서 얻음"이라 직관적이고 부익부 가속(빠른 결판)이지만:
  ① 당첨자 의미론 플립(최소 가해자 = 당첨) — 튜닝 자산·직전 리프레이밍과 단절
  ② 다운자는 가해 0 → 탈출에서 멀어지는 음의 나선(다운 페널티 이중화)
  ③ Codex도 "B가 반드시 낫다는 증거 없음 — 블라인드 관전 테스트로만 검증 가능" 판정.
- **재평가 트리거**: 적용 후 플레이테스트에서 "왜 저 사람이 당첨인지 모르겠다"(인과 가독성) 피드백이 재발하면 B안을 High로 격상해 재검토한다.

### C안 (기각) — 메커니즘 무변경 + 종료 압축/HUD만
- 최소 변경(Effort S)이지만 P2(능동 승리 구조 = goal 핵심 요구)를 해소하지 못함 — goal의 ①②④ 중 ①을 버리는 안이라 탈락.

## 3. 채택 설계 상세

### 3-1. 핵심 수치 (공유 상수 — 서버/클라 동시 수정)

| 상수 | 초기값 | 의미 | 권위 |
|------|--------|------|------|
| `BLADE_COUNT` | 2 (유지) | 시작 칼 수 | 고정 |
| `ESCAPE_BLADES` | 5 | 이 개수 도달 = 탈출 (기존 상한 5 재활용) | 고정(UI 문구 결합) |
| `BLADE_UP_DMG` (T) | 45 → **최종 35** (배치 확정) | 받은 데미지 누적 임계당 칼 +1 (5−2 = 3회 필요 = 총 105) | **200시드 배치가 권위 — 탐색 범위 30~55. T=45/40은 h2 결판률 91%/90.5% FAIL(2인 접촉 빈도 병목), T=35에서 h2 97.5%·h3 99.5%·h4~6 100% 전부 PASS** |
| `REVIVE_MS` | 3000 | 다운 → 부활 시간 (goal 고정) | 고정 |
| `REVIVE_GRACE_MS` | 800 | 부활 직후 다운 면역(피해·진행은 정상, HP만 1 미만 불가 클램프) | 배치 권위 |
| `DECIDE_TAIL_MS` | 2000 | decideMs 후 결판 비트 길이 (30초 캡에 잘리면 잘린 만큼만 — 캡 우선) | 고정 |
| `GAME_MS` | 30000 (유지) | 교착 시 상한 캡 | 고정 |
| `HP_MAX` / `HIT_DPS` | 100 / 110 (유지) | 체력/접촉 데미지 | 배치서 필요시만 조정 |

- T=45 선택 근거: 100과 비배수(칼업과 다운이 같은 틱에 겹치는 빈도 감소), 3회 = 135 ≈ 현행 첫 킬(100) 대비 1.35배. h4~6은 다운 3초간 그 사람 칼날이 사라져 상호 진행 정체가 생기므로 첫 배치에서 T=45가 크게 빗나갈 수 있음을 예상 — **배치 분포가 기준 미달이면 T부터 30~55 구간 탐색, 평탄 T로 h2~h6 동시 충족 실패 시에만 인원수별 스케일(fallback 노브) 도입.**
- grace 클램프의 회계 영향: 클램프 발동 중에도 cumDmg(받은 데미지 누적)는 정상 적산된다. **진행도 표시는 hp가 아니라 cumDmg 채널(3-3)에서 파생하므로 표시 왜곡 없음.**

### 3-2. 틱 내 이벤트 순서 (결정론 명문화 — 2탭 동기 핵심)

매 틱(SIM_DT_MS=20ms) 순서를 고정한다. 클라는 이 순서를 재현하지 않고 **서버가 내려준 이벤트(timeMs)만 소비**한다.

1. 링 반경·칼날 선분 스냅샷 — **활성**(탈출 아님 + 다운 아님) 캐릭터의 칼날만 생성. 링 수축 스케줄(10~20s 절대 시간)은 **가변 길이와 무관하게 유지** — 결판이 빠른 판은 수축 전에 끝나도 무방(수축은 교착 방지 압력 장치).
2. 데미지 적산: 활성 캐릭터만 피격 대상. `cumDmg += 받은 데미지`, `hp −= 동일량`(grace 중엔 hp만 1 미만 불가 클램프, cumDmg는 정상), 넉백 임펄스 동일. **decideMs 확정 후에는 피격 판정 자체를 스킵(데미지·넉백·cumDmg 전부 없음)** — 결판 비트는 이동/부활 타이머만 진행.
3. **칼업 후보 산출**: `while (cumDmg >= 다음임계 && bladeCount < 5)`로 칼업을 적용하되, **5번째 칼(=탈출 자격)에 도달하는 칼업은 즉시 push하지 않고 "탈출 후보"로 보류**한다. 4개 이하 칼업은 `bladeUps.push({id, timeMs})` 즉시 확정 (한 틱 다중 칼업 허용).
4. **탈출 처리**: 탈출 후보들을 `cumDmg` 내림차순 → slotId 오름차순으로 순회. 각 후보에 대해 — 탈출시켜도 아레나 잔류자(**다운 중 포함** — "탈출 안 한 사람" 전부)가 1명 이상 남으면: 5번째 bladeUp push + `escapes.push({id, timeMs, x, y})` + 좌표 동결·시뮬 이탈. **잔류자가 1명이 되는 순간 남은 후보는 전부 탈출 거부**(항상 최소 1명 잔류 규칙 — "문이 닫혔다"): **5번째 bladeUp을 발행하지 않고 칼 4개로 남는다**(핍 4/5 + 적색 — goal "진행도 안 찬 채 강조"와 정합). 같은 틱에 HP 0과 탈출 자격 동시 도달 시 탈출이 우선(탈출 성공자는 다운 체크 면제).
5. **다운 체크**: 활성(이번 틱 탈출자 제외) + `hp <= 0` → 다운: 그 자리 좌표 동결, `downs.push({id, timeMs, reviveMs: timeMs+3000, x, y})`. grace 중에는 hp 클램프로 여기 도달 불가. 탈출 거부자(칼 4 잔류)도 정상 다운 가능(막판 KO 서사).
6. **부활 체크**: `t >= reviveMs` → 활성 복귀, `hp = 100`, `graceUntil = t + 800`, 동결 좌표 그대로 재개(칼 수·cumDmg 유지). 부활 좌표가 수축한 링 밖이면 **같은 틱 ⑦의 하드 월 클램프로 즉시 재진입**(비석 표시 좌표는 동결값 유지 — 시체처럼 링 밖에 남아도 무방).
7. 이동 적분(드리프트+넉백+중앙 인력+링 하드 월) — 활성 캐릭터만. 다운/탈출자는 좌표 동결.
8. **결판 체크**: 잔류자(탈출 안 한 사람) 수 == 1 → `decideMs = t` (1회). 시뮬은 `durationMs`(아래 라운딩 규칙)까지만 진행.

- **durationMs 라운딩 규칙(필수)**: `durationMs = min(GAME_MS, ceil((decideMs + DECIDE_TAIL_MS) / SAMPLE_MS) * SAMPLE_MS)` — decideMs는 20ms 격자라 SAMPLE_MS(100) 배수 보장이 없으므로 올림 정렬. `frames.length === durationMs/SAMPLE_MS + 1` 불변식이 이 규칙으로 성립.
- 기존 "decided → 최후 1인 무적" 규칙은 자연 소멸(혼자 남으면 데미지 원천 0) — 명시 코드는 "decideMs 후 피격 판정 스킵"으로 대체.
- 30초 캡 도달(결판 못 냄): `decideMs = null`, `durationMs = 30000`, selected = 진행도 최하위(3-4).

### 3-3. reveal 페이로드 (구조 변경 — 같은 배포로 서버/클라 동시 교체)

```
{
  durationMs,            // 가변! 3-2 라운딩 규칙 산출값. (frames.length-1)*SAMPLE_MS와 항상 일치
  decideMs,              // 결판 시각(ms) 또는 null(캡 종료) — 클라 결판 연출 기준
  sampleMs, arena, ring, slots,   // 기존 유지 (slots에 bladeCount=2 시작값 유지)
  frames,                // [x, y, cumDmg]*n 키프레임 — 길이 가변. ★3번째 채널이 hp→cumDmg(받은 데미지 누적, 반올림 정수)로 교체★
  escapes:  [{id, timeMs, x, y}],          // 탈출 이벤트 (rank = 배열 순서+1)
  downs:    [{id, timeMs, reviveMs, x, y}],// 비석 + 3·2·1 + 부활 연출 기준 (슬롯당 다회 가능)
  bladeUps: [{id, timeMs}],                // 핍 HUD/칼날 렌더 개수의 유일 권위
  result: { selected, rankings }           // rankings 스키마는 3-4
}
```

- **frames hp→cumDmg 교체 사유(스펙 리뷰 F-1)**: HP UI가 완전 제거되어 hp 채널의 유일 소비자는 진행 바인데, grace 클램프·다운·부활 리셋 때문에 hp로는 cumDmg를 정확 파생할 수 없다. cumDmg는 단조 증가라 ① 진행 바 = `(cumDmg − T×(bladeCountAt−2)) / T` (clamp 0~1) 단순 파생 ② 피격 연출 감지 = "cumDmg 상승 중"(기존 hpDrop과 동일 역할, 부활 점프 오감지 없음) ③ 데미지 숫자 = 키프레임 간 cumDmg 증가량 — 전부 해결. 페이로드 폭 불변(n×3).
- `eliminations` 필드는 **제거**(escapes/downs로 대체). savedReveal은 메모리 전용·페이로드 미영속이라 안전, 배포 중 구버전 캐시는 캐시버스트(v++)로 해소.
- 클라 `bladeCountAt(si, t)` = `2 + count(bladeUps where id===si && timeMs < t)` (strict `<` 기존 관례 유지 — 탈출 순간 t=timeMs에 핍이 4/5로 보이는 1틱 지연은 의도된 동작, 다음 키프레임에 5/5). killerId 파생 로직(_killTimes) 제거.
- `finalState`(server-only)에 `bladeCount`/`cumDmg` 포함 — rankHumans와 테스트가 사용.
- 새 필드(escapes/downs/bladeUps/decideMs)도 **reveal 시점에만 공개** — `socket/rooms.js` 재진입 마스킹(timeline/result/seed 제외) 계약 유지.

### 3-4. 순위/당첨 (rankHumans 재작성)

- rankings 엔트리 스키마: `{ name, slotId, rank, escapeMs }` — `escapeMs` = 탈출 시각(ms), 탈출 못 한 사람은 `null`. (기존 `eliminatedMs` 필드명 대체 — 클라 결과 라벨도 함께 갱신.)
- rank 1 = 첫 탈출, … rank n−1 = 마지막 탈출, rank n = selected(잔류자).
- 30초 캡 교착(잔류 2+명): 잔류자끼리 `bladeCount` 내림차순 → `cumDmg` 내림차순 → slotId 오름차순. **selected = 정렬 최하위**(진행도가 가장 낮은 사람, 전부 동률이면 slotId 큰 쪽).
- **캡 교착 표시 규칙**: selected만 적색 `못 나감`/`⚔️ 탈출 실패 (당첨)`. 비당첨 잔류자는 패널 `칼 k/5` 유지 + 결과 오버레이 `✅ 통과`(기존 중립 태그 재사용).
- selected의 결과 라벨 분기: 종료 시점에 다운 상태(`∃ down: timeMs ≤ durationMs < reviveMs`)면 `⚔️ 막판 KO (당첨)`, 아니면 `⚔️ 탈출 실패 (당첨)` — 클라가 downs에서 파생.
- 동일 시드의 당첨자가 기존 시뮬과 달라지는 것은 **의도된 변화**(시뮬 자체가 새것) — 직전 리프레이밍의 "동일 인물 보장" 제약은 이번엔 적용되지 않는다.
- DB 시맨틱 무변경: selected = isWinner false/rank 2, 나머지 rank 1. `gameRules: 'spin-survival'` 유지.

### 3-5. 종료 압축 (서버/클라)

- 서버: `sa.endTimeout = setTimeout(endGame, COUNTDOWN_MS + durationMs + RESULT_HOLD_MS)` — durationMs는 시뮬 산출(시드 결정론). **서버/클라 동시 교체 필수**(어긋나면 gameEnd가 리플레이 도중/한참 뒤 도착).
- 클라: 핵심 루프(t 클램프 :876, nearEnd :1059, 종료 분기 :1208)는 이미 `payload.durationMs` 기준(Scout 검증 — 클라 GAME_MS/SAMPLE_MS/COUNTDOWN_MS는 선언만 있고 사용 0회). `nearEnd(마지막 2초)` 글로우가 자동으로 결판 비트에 정렬된다.
- decideMs > 28s면 결판 비트가 2초 미만으로 잘림 — **캡 우선, tail 잘림 허용**.
- 안전장치: 교착 시 30초 캡(현행 길이 유지). 잔류자가 다운 상태로 결판 나는 엣지 = 비석 상태로 적색 강조(`막판 KO` 라벨과 일치), 결판 비트 동안 부활 타이머는 정상 진행(혼자 일어나는 모습 = 못 나감 서사).

### 3-6. HUD — 채택: (a) 칼 핍 + (c) 미션 패널 (goal "택1~2" 충족 / (d) 임박 강조는 (a)의 저비용 부속 효과로 포함)

- **캔버스 머리 위**: 기존 30px HP바 자리에 `◆◆◇◇◇` 핍 5칸(채움 = bladeCountAt) + 그 아래 얇은 진행 바. **진행 바 공식**: `clamp((cumDmg − (bladeCountAt(si,t)−2) × T) / T, 0, 1)` — 칼업 순간 리셋되며 다시 차오르는 게 정상 문법(P1 해소). HP 표시는 **완전 제거**(다운은 비석이 말해줌 — 관전형이라 체력 정밀도 불필요).
- **칩 잔상(chipHp)은 제거** — 진행 바는 잔상 없는 단순 채움. cumDmg 단조 채널에선 하락 수렴 잔상이 무의미하고, P1이 지목한 "다시 차는" 인상의 원인 절반이 바로 이 하이라이트다 (`js/spin-arena.js:908~910` 로직 삭제).
- **탈출 차단자(최소 1명 잔류 규칙로 칼 4 고정)의 바 표시**: 진행 바 **적색 고정(가득)** + 핍 4/5 — "문 앞에서 닫혔다" 서사를 명시 표현. goal의 "안 찬 채 강조"는 핍(미션 지표)이 4/5로 충족.
- **미션 패널**(기존 HP 패널 전환): 행 = 색점 + 이름 + `칼 3/5` + 상태(`전투중` / `🪦 3·2·1` / `✓ k위 탈출` / 결판·캡 시 selected만 `못 나감` 적색, 비당첨 잔류자는 `칼 k/5` 유지).
- **임박 강조**: 칼 4개(N−1) 도달 시 캐릭터 글로우 펄스 + 기존 사운드 재사용 — "곧 탈출" 텐션.
- **미션 텍스트**: 카운트다운 백드롭 중앙 + gameStatus에 `⚔️ 칼 5개를 모으면 탈출!` 고정 안내. 튜토리얼/결과 라벨과 문구 통일.
- **부활 표시**: 비석은 캔버스 프로시저럴 드로잉(둥근 상단 회색 비석 + 음각 — 이모지 폰트 의존 회피), 머리 위 3·2·1 카운트다운(t 파생), 부활 순간 플래시+링(기존 fx 재사용).
- (b) 원형 진행 링은 기각 — 핍과 정보 중복, 회전 칼날과 시각 간섭.
- 신규 리소스: **이미지 0, 사운드 0** (전부 기존 fx/사운드 키 재사용 — 키 목록: start/hit/eliminate/result/bgm. 다운=`spin-arena_eliminate` 저볼륨, 탈출=`spin-arena_eliminate` 기존 그대로, 부활/칼업=`spin-arena_hit` 계열 재사용. 전용 사운드는 후속 과제로 기록만).

### 3-7. 연출 매핑 (기존 자산 재사용)

| 이벤트 | 연출 |
|--------|------|
| 칼업(bladeUp) | 기존 `⚔️+1` 플래시 재사용(칼날 스냅 마스킹 기능 동일 — 트리거만 kill→bladeUp으로, `bladeFlashDone` 키도 bladeUps 인덱스로 교체) |
| 탈출(escape) | 기존 탈출 연출 그대로(파편+링+플래시+`탈출!` 골드 텍스트+상승 페이드 퇴장 700ms) |
| 다운(down) | 회색 시체 대신 **비석 드롭**(스케일 펀치) + `🪦` 패널 상태 + 흔들림 |
| 부활(revive) | 플래시 + 스킨색 링 + 3·2·1 텍스트 소멸 |
| 결판(decideMs) | 잔류 1인 적색 글로우/⚠️(기존 nearEnd 로직이 durationMs 가변화로 자동 정렬) |
| 결판 줌 | `showdownStartT` 재정의: n≥3이면 "잔류 2명 진입 시각"(= (n−2)번째 escape의 timeMs), **n=2는 decideMs**(시작 즉시 줌 방지 — 스펙 리뷰 C-5). decideMs가 null(캡)이면 줌 생략 |

- **클라 fx 캐시 구조 변경 주의(스펙 리뷰 F-3)**: 기존 `_elimMs`/`elimX/Y`는 슬롯당 단일 값 — downs는 슬롯당 다회라 **슬롯별 downs 배열 + 구간 판정**(`다운 중 = ∃k: downs[k].timeMs ≤ t < downs[k].reviveMs`)으로 재설계. `_killTimes`→bladeUps 재배선, `burstDone`→escape/down별 1회 트리거 키 재설계, `chipHp` 로직 삭제(3-6) 포함. 과소 견적 금지.

### 3-8. 공정성 (절대 불변)

- 결과는 서버 결정론 시뮬에서만(시드 PRNG, 동일 시드 = 동일 frames/escapes/downs/bladeUps/decideMs/result).
- 클라 Math.random 결과 영향 0 (deviceId/tabId 2회 유지). 모든 신규 연출 = t·페이로드 파생.
- reveal 전 server-only(timeline/result/seed + 새 이벤트 필드) 비노출·재진입 마스킹 유지.

## 8. 막힘/후퇴 기준 (goal 승계)

- 배치 분포 기준 미달이 T(30~55) 조정으로 해소 안 되면: 수치 조합 탐색 결과를 표로 보고 후 가장 근접한 조합 채택 + 근거 보고.
- durationMs 가변화가 클라 길이 의존부를 깨면: frames 절단을 포기하지 말고 **그 의존부를 수리**(durationMs 단일 권위화 — Scout 조사로 핵심 루프는 이미 안전 확인). 수리가 과대 작업이면 frames는 30초 풀 길이 유지 + 클라가 durationMs까지만 재생하는 방식으로 후퇴.
- 관전 인과 가독성 피드백 재발 시: B안(가한 데미지) High 격상 재검토 (2절 트리거).

## 9. Success Criteria (goal 완료 기준과 동일)
