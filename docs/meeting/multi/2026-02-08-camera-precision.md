# LAMDiceBot 팀 회의록

**일시**: 2026-02-08
**주제**: 경마 카메라 무빙 시스템 정교화
**참석자**: 기획자, 개발자, QA/UX
**회의 방식**: 3인 멀티에이전트 독립 분석 → 교차 검토 → 합의 도출

---

## 1. 현황 요약

- 카메라: 고정 lerp(0.08), 랜덤 컷어웨이(3초 고정), 단순 패닝
- 카메라 모드 3가지: 리더 추적, 내 말 보기, 꼴등 추적
- 1등 도착 → 500ms 대기 → 2500ms 패닝으로 꼴등 이동
- 꼴등 기준 버그: 슬로우모션은 베팅 기준, 카메라는 전체 기준 (불일치)

## 2. 독립 분석 결과

### 2-1. 기획자 (A)
1. **동적 Lerp 속도** (상) - 빠른 장면에서 카메라 즉각 반응
2. **드라마틱 모멘트 자동 감지** (상) - 접전/역전/꼴등 경쟁 자동 인식
3. **카메라 전환 이펙트 다양화** (중) - 스냅/팬/줌/쉐이크 등
4. **사용자 설정 기능 확장** (중) - 컷어웨이 빈도, 속도 프리셋
5. **결승선 도착 연출 강화** (상) - 1등→전체뷰→꼴등 4단계 시퀀스

### 2-2. 개발자 (B)
1. **적응형 Lerp Speed** (하) - 거리 기반 lerp 동적 조정
2. **경기 상황 인식 컷어웨이** (중) - CLOSE_RACE/RUNAWAY/EARLY_STAGE 분류
3. **역전/접전 자동 감지 하이라이트** (중) - HighlightDetector 클래스
4. **requestAnimationFrame 마이그레이션** (하) - setInterval → RAF
5. **Dead Code 제거 + 모듈화** (상) - CameraController 클래스

### 2-3. QA/UX (C)
1. **적응형 카메라 속도** (중) - 부스트/접전/꼴등 상황별 lerp
2. **지능형 컷어웨이** (중) - 순위 가중치 + 예고 텍스트
3. **접전 감지 스플릿 뷰** (상) - 와이드 앵글 + 접전 표시
4. **카메라 모드 전환 시각 피드백** (하) - 전환 애니메이션 + HUD
5. **모바일 결승선 가시성 최적화** (중) - 동적 오프셋 + 미니맵

## 3. 교차 검토 결과

### 3-1. 기획자 검토
- B1(적응형 lerp): ROI 높음 ✅
- B3(하이라이트): 게임 정체성 강화 → 최우선 ✅
- C1(적응형 속도): B1보다 구체적 ✅
- C2(지능형 컷어웨이): 꼴등 경쟁자 우선 표시 → 최우선 ✅
- C3(스플릿뷰): 꼴등 게임과 부합도 낮음, 리스크 대비 효과 낮음 → 보류
- C4(전환 피드백): 퀵윈 ✅

### 3-2. 개발자 검토
- A1(동적 lerp): 구현 쉬움, 즉시 적용 가능 ✅
- A3(이펙트 다양화): 줌/쉐이크 성능 저하 위험 → 제한적
- A5(결승 연출): 기존 패닝 확장으로 가능 ✅
- C3(스플릿뷰): 멀티 뷰포트 필요 → 아키텍처 재설계 불가 ❌
- C4(전환 피드백): 구현 간단 ✅

### 3-3. QA/UX 검토
- A2(드라마틱 감지): 과도한 전환 → 어지러움 위험 ❌
- A5(결승 연출): 베팅 게임 핵심 UX → 매우 긍정 ✅
- B1(적응형 lerp): 거리 기반이라 예측 가능하고 테스트 용이 ✅
- B4(RAF): 필수 개선 ✅ (단 이번 범위에선 보류)
- B5(리팩토링): 장기 과제, 별도 스프린트 배정 권장
