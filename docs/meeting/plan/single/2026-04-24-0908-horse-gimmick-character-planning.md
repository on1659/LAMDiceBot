# 회의록: 경마 신규 기믹 및 캐릭터 배경 확장
> 일시: 2026-04-24

## 안건

경마에 새 기믹이나 캐릭터 배경을 추가한다면 무엇이 좋은지 검토한다.

현재 경마는 서버 시뮬레이션 기반으로 순위를 확정하고, 클라이언트는 이를 재생한다. 이미 개별 말 기믹(stop, slow, sprint, reverse 등), 날씨 보정, 슬로모션, 진화 기믹, 탈것별 테마 배경, 탈것 통계가 존재한다. 따라서 다음 확장은 "결과 공정성은 유지하면서 관전 몰입과 선택의 재미를 키우는 방향"이 가장 적합하다.

## 기획 검토

### 현우 (리서치)

- 대상 사용자: 친구들과 방을 만들어 짧은 랜덤 승부를 즐기는 호스트/참여자, 모바일로 관전하는 사용자, 결과보다 과정의 드라마를 보는 사용자
- JTBD: "내가 고른 탈것이 단순 번호가 아니라 캐릭터처럼 느껴지고, 경주마다 볼거리가 조금씩 달라졌으면 좋겠다"
- MoSCoW:
  - Must: 캐릭터 배경 카드, 탈것별 한 줄 성격, 결과에 영향 없는 해설/칭호
  - Should: 트랙 존 시스템, 날씨/트랙과 캐릭터 반응 문구
  - Could: 라이벌 매치, 탈것별 미니 업적, 해설 아나운서 스타일 선택
  - Won't: 실시간 방해 클릭, 순위 셔플, 유료/포인트 배팅
- 성공 지표: 말 선택 화면 체류 시간 증가, 다시보기 클릭률 증가, 동일 방 연속 경주 수 증가, 채팅 반응 증가

### 소연 (전략)

- KPI 연결: 경마의 체류 시간과 반복 플레이를 늘리는 것이 핵심이다. 캐릭터 배경은 구현 비용 대비 재방문/몰입 효과가 크고, 트랙 존은 게임 장면의 공유 순간을 만든다.
- ICE 점수:
  - 캐릭터 배경 카드: I=8 x C=9 x E=9 = 648
  - 트랙 존 시스템: I=9 x C=8 x E=6 = 432
  - 라이벌 매치: I=7 x C=7 x E=7 = 343
  - 실시간 인터랙션: I=8 x C=5 x E=3 = 120
- 수익 영향: 페이지 체류 시간이 늘면 광고 노출 기회가 증가한다. 단, 레이스 화면을 가리는 광고/모달식 기능은 금지한다.

## 구현 검토

### 태준 (BE)

- 수정 파일:
  - `assets/vehicle-themes.json`: 캐릭터 설명, 성격 태그, 라이벌, 별명 필드 추가
  - `utils/vehicle-helpers.js`: 서버에서 이름/메타데이터 재사용 시 공통 매핑 확장
  - `socket/horse.js`: 트랙 존, 라이벌 매치, 레이스 기록 확장 시 수정
  - `config/horse/race.json`: 트랙 존 설정 추가 시 수정
- DB 영향:
  - v1 캐릭터 배경 카드는 DB 변경 없음
  - 탈것별 업적/호감도/성장 시스템은 DB 필요하므로 v2 이후
- Socket 영향:
  - 캐릭터 배경만 표시하면 기존 `selectedVehicleTypes`로 클라이언트가 로컬 메타데이터를 lookup 가능
  - 트랙 존은 `horseRaceCountdown`, `horseRaceStarted`, `record`에 `trackZones` 추가 필요
- 예상 공수:
  - 캐릭터 배경 카드: 0.5~1일
  - 트랙 존 시스템: 2~3일
  - 라이벌 매치: 1~2일
- 리스크:
  - 결과에 영향을 주는 기능은 반드시 서버 사전 계산에 포함해야 한다
  - 캐릭터 설명이 실제 성능처럼 읽히면 공정성 오해가 생긴다

### 미래 (FE)

- 수정 파일:
  - `js/horse-race.js`: 말 선택 카드 확장, 결과/히스토리 문구 표시, 트랙 존 렌더링
  - `css/horse-race.css`: 캐릭터 카드/태그/트랙 존 스타일
  - `horse-race-multiplayer.html`: 필요 시 도움말 문구 추가
  - `js/horse-race-commentary.js`: 캐릭터/라이벌 해설 추가
- CSS 변수:
  - 캐릭터 카드용 별도 색상은 최소화하고 기존 `--horse-*`, `--result-*`, `--status-*` 재사용
  - 트랙 존은 `--zone-mud`, `--zone-ice`, `--zone-boost` 정도만 추가
- 화면 대응:
  - 모바일(375px): 말 선택 카드에는 이름, 별명, 1줄 성격만 노출. 상세 배경은 탭/확장 버튼으로 표시
  - 데스크톱(1920px): 선택 카드 hover/focus 시 캐릭터 배경, 강점 플레이버, 라이벌 정보를 옆 패널로 표시
  - 브레이크포인트 전환: 모바일은 카드 높이 고정, 데스크톱은 그리드 2~3열 유지
- 예상 공수: 캐릭터 카드 1일, 결과/히스토리 문구 0.5일, 트랙 존 렌더링 1~2일

## 디자인 검토

### 다은 (UI)

- CSS 변수: 기존 경마 컬러가 갈색 계열로 강하므로 캐릭터 카드마다 과한 고유색을 주기보다 작은 태그와 아이콘 중심으로 차이를 준다.
- 레이아웃: 선택 화면에서 탈것을 "성능표"처럼 보이게 만들지 말고, 캐릭터 도감 느낌의 짧은 정보로 둔다.
- 상태 피드백:
  - 선택됨: 기존 선택 테두리 + 캐릭터 별명 강조
  - 인기 탈것: 기존 인기 배지 유지
  - 라이벌 성립: 두 탈것이 함께 출전할 때만 작은 VS 표시
- 모바일 UI:
  - 터치 타겟 44px 이상 유지
  - 카드 내부 텍스트는 2줄까지만 표시
  - 긴 배경 설명은 레이스 시작 전이 아니라 선택 화면에서만 접근 가능하게 한다

### 승호 (UX)

- 사용자 플로우:
  - 호스트가 방 생성
  - 참여자가 탈것 선택
  - 카드에서 "이 탈것은 어떤 애인지" 짧게 확인
  - 레이스 중 해설이 캐릭터/라이벌을 언급
  - 결과 화면에서 승리/패배 칭호가 표시
- 마찰 포인트:
  - 설명이 길면 선택 속도가 느려진다
  - 능력치처럼 보이면 "이 탈것이 유리한가?"라는 오해가 생긴다
  - 랜덤 선택 사용자는 정보를 놓칠 수 있다
- 접근성:
  - 색상만으로 캐릭터 타입을 구분하지 않는다
  - 아이콘 옆에 텍스트 태그를 함께 표시한다
  - 레이스 중 해설은 사운드 없이도 화면 텍스트로 확인 가능해야 한다
- 모바일 UX:
  - 선택 전 상세 보기보다 선택 후 짧은 확인 문구가 적합하다
  - 네트워크 지연 중에도 캐릭터 카드가 빈 상태로 보이지 않도록 로컬 JSON fallback 필요

## PD 최종 판단

### 지민 (PD)

- 판정: 조건부 Go
- 범위: v1은 "캐릭터 배경 카드 + 결과 칭호 + 캐릭터 해설"까지. v2는 "트랙 존 시스템". v3는 "라이벌 매치/업적".
- 배포 리스크: v1은 낮음. v2부터 서버 시뮬레이션과 클라이언트 재생 동기화 테스트 필요.
- 담당자:
  - v1 데이터/문구: 기획
  - v1 UI/해설: FE
  - v2 트랙 존: BE + FE + QA

### 예상 사용 시나리오

- As-Is: 플레이어는 탈것을 고르지만, 탈것은 사실상 이름과 이미지 중심이다.
- To-Be: 각 탈것에 별명, 성격, 배경, 결과 칭호가 붙어 "내가 고른 캐릭터" 느낌이 생긴다.
- 사용 흐름: 선택 화면에서 캐릭터를 보고 고름 → 레이스 중 해설이 캐릭터성을 언급 → 결과 화면에서 승리/패배 칭호로 마무리
- 기대 효과: 선택 단계가 더 즐거워지고, 경주마다 이야기거리가 생기며, 채팅 반응과 다시보기 동기가 늘어난다.

## 추천 구현 순서

### 1순위: 탈것 캐릭터 배경 카드

결과에 전혀 영향을 주지 않고 몰입만 키우는 가장 안전한 확장이다.

예시 데이터 필드:

```json
{
  "horse": {
    "title": "정통파 질주마",
    "personality": "묵묵히 달리다가 마지막에 표정이 바뀐다",
    "tags": ["정석", "근성", "클래식"],
    "winLine": "역시 기본기가 답이었다.",
    "loseLine": "오늘은 말발굽이 조금 무거웠다."
  }
}
```

초안 캐릭터 방향:

| 탈것 | 별명 | 성격 |
|------|------|------|
| 자동차 | 고속도로의 현실주의자 | 꾸준하고 시끄럽게 자신감 있음 |
| 로켓 | 카운트다운 중독자 | 느리면 자존심 상하는 폭발형 |
| 새 | 바람 읽는 정찰병 | 조용히 있다가 갑자기 치고 나감 |
| 보트 | 물길 없는 항해사 | 트랙이 뭐든 파도처럼 탄다 |
| 자전거 | 페달 장인 | 작지만 끈질긴 체력파 |
| 토끼 | 출발선의 스타 | 초반 쇼맨십이 강함 |
| 거북이 | 느긋한 계산가 | 늦어 보여도 끝까지 감 |
| 독수리 | 하늘의 심판관 | 높은 곳에서 판세를 읽음 |
| 킥보드 | 골목길 스피드광 | 민첩하지만 가끔 과감함 |
| 헬리콥터 | 소음 많은 구조대 | 위기 때 더 존재감 있음 |
| 말 | 클래식 챔피언 | 이름값을 지키려는 정통파 |
| 기사 | 왕국 대표 선수 | 명예와 포즈를 중시함 |
| 공룡 | 오래된 피지컬 | 느릿하지만 충돌감이 강함 |
| 닌자 | 그림자 추월자 | 조용히 숨어 있다가 나타남 |
| 게 | 옆걸음 전략가 | 정면 승부인 척 이상하게 감 |

### 2순위: 결과 칭호와 레이스 후 한 줄 평

결과 화면과 히스토리에 짧은 문구를 붙인다. 순위 계산에는 영향이 없다.

예시:

| 조건 | 칭호 |
|------|------|
| 1등 | 오늘의 주인공 |
| 꼴등 | 드라마 담당 |
| 진화 발동 후 순위 상승 | 각성 성공 |
| 초반 하위권에서 3위권 진입 | 역전 서사 |
| 배팅 안 된 탈것 | 쓸쓸한 완주자 |

### 3순위: 캐릭터 기반 해설 확장

`js/horse-race-commentary.js`에 범용 해설과 캐릭터별 해설을 분리한다.

예시:

```js
const VEHICLE_COMMENTARY = {
  rocket: ['로켓이 발사각을 다시 잡습니다!', '관제탑에서 소리 지르는 중입니다!'],
  turtle: ['거북이가 천천히 판을 읽고 있습니다.', '느린 게 아니라 오래 가는 겁니다.'],
  ninja: ['닌자가 화면 밖에서 추월각을 봅니다.']
};
```

### 4순위: 트랙 존 시스템

기존 개별 기믹과 다른 차원의 확장이다. 모든 말에게 동일한 구간 효과를 적용하면 공정성을 유지하면서 경주마다 다른 맵 느낌을 줄 수 있다.

추천 v2 존:

| 존 | 효과 | 비고 |
|----|------|------|
| 진흙 | 구간 내 감속 | 가장 직관적 |
| 빙판 | 약한 감속 + 미끄럼 연출 | 시각 재미 |
| 부스터 패드 | 구간 내 가속 | 역전 포인트 |

### 5순위: 라이벌 매치

같은 방에 특정 조합이 등장하면 결과 영향 없이 해설과 VS 배지만 표시한다.

예시:

| 조합 | 관계 |
|------|------|
| 토끼 vs 거북이 | 고전 라이벌 |
| 로켓 vs 헬리콥터 | 하늘 소음 대결 |
| 기사 vs 닌자 | 명예와 은밀함 |
| 공룡 vs 자동차 | 과거와 현재 |
| 보트 vs 게 | 해변 대표전 |

결과 영향 보너스는 v1에서 제외한다.

## 최종 추천

바로 시작할 기능은 **캐릭터 배경 카드 + 결과 칭호 + 캐릭터 해설**이다.

이 조합은 공정성을 건드리지 않고도 경마의 체감 재미를 크게 올린다. 그 다음 단계로 **트랙 존 시스템**을 넣으면, 현재의 개별 기믹 중심 경마가 "캐릭터가 있는 레이스 + 매번 다른 코스"로 확장된다.
