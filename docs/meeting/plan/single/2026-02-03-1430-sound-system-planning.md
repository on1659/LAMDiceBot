# LAMDiceBot 팀 회의록 (Single)

**일시**: 2026-02-03
**주제**: 사운드 시스템 기획 - 경마/주사위/룰렛 게임별 효과음 및 BGM 선정
**참석자 (관점)**:
- 기획팀: 기획 팀장, 기획 사원
- 개발팀: 개발 팀장, 개발 사원
- 디자인팀: UI/UX 디자이너, 사운드 디자이너
- 품질팀: QA 엔지니어
**회의 방식**: 1인 순차 분석 (경량 버전)
**게임 대상**: 주사위 / 룰렛 / 경마 / 팀 배정 / 공통

---

## 1. 현황 요약

### 사운드 시스템 인프라
- **SoundManager** (`assets/sounds/sound-manager.js`): 구현 완료
  - `playSound(key)`: 단발 효과음
  - `playLoop(key)`: 루프 BGM
  - 탭 포커스 체크, 볼륨 조절 지원
- **설정 파일**: `assets/sounds/sound-config.json` (18개 키 등록)

### 에셋 현황

| 게임 | 전체 | 실제 에셋 | 교체 필요 |
|------|------|-----------|-----------|
| 주사위 | 4 | 1 (roll) | 3 |
| 룰렛 | 4 | 3 (spin, stop, winner) | 1 (bgm) |
| 경마 | 6 | 0 | 6 |
| 팀배정 | 1 | 0 | 1 |
| 공통 | 3 | 1 (button) | 2 |
| **합계** | **18** | **5** | **13** |

### 핵심 문제
1. **경마 게임**: 사운드 0개 — 가장 시급
2. **BGM 전체 부재**: 모든 게임에 BGM 없음
3. **에셋 소싱**: 어디서 어떤 사운드를 구할 것인가?

## 2. 관점별 분석 결과

### 기획팀

#### 2-1. 기획 팀장 (A1)

**전략적 우선순위**

1. **경마 사운드 최우선**: 경마가 v2.4 신규 기능이므로 사운드 완성이 비즈니스 임팩트 큼
   - 효과음만으로도 몰입감 300% 상승 기대
   - BGM은 차후 추가 가능

2. **무료 에셋 전략 권장**: 저작권 리스크 회피
   - freesound.org, pixabay.com 등 CC0 라이센스 활용
   - 상업적 사용 시에도 안전

3. **게임별 사운드 정체성 구분**
   - 주사위: 가볍고 캐주얼
   - 룰렛: 카지노 느낌, 긴장감
   - 경마: 스포츠 중계, 역동적

#### 2-2. 기획 사원 (A2)

**사용자 경험 관점**

1. **필수 효과음 TOP 5** (유저 반응 예상 기준)
   - 🥇 경마 출발 총소리 — "시작이다!" 느낌
   - 🥈 경마 관중 환호 — 레이스 중 긴장감
   - 🥉 경마 1등 골인 팡파레 — 보상감
   - 4️⃣ 주사위 결과 확정음 — "띵!" 피드백
   - 5️⃣ 룰렛 당첨자 발표음 — 이미 있지만 강화 가능

2. **BGM 필요성 재검토**
   - 짧은 게임(주사위, 팀배정)에는 BGM 불필요할 수 있음
   - 경마(레이스 시간 있음)에는 BGM이 몰입감에 기여

3. **사운드 ON/OFF 기본값**
   - 현재 기본 ON인데, 모바일에서는 기본 OFF가 UX에 유리할 수 있음
   - "첫 터치 시 사운드 활성화" 패턴 제안

---

### 개발팀

#### 2-3. 개발 팀장 (B1)

**아키텍처 관점**

1. **현재 SoundManager 구조 양호**
   - 키 기반 추상화로 확장 용이
   - config.json만 수정하면 에셋 교체 가능
   - 기술 부채 없음

2. **에셋 용량 관리 필요**
   - 각 mp3 파일 500KB 이하 권장 (모바일 로딩)
   - 필요시 압축률 높은 OGG 포맷 병행 고려

3. **Preload 전략 추가 제안**
   - 게임 시작 전 해당 게임 사운드 미리 로드
   - 첫 재생 시 딜레이 방지

#### 2-4. 개발 사원 (B2)

**구현 관점**

1. **경마 사운드 트리거 포인트 정리**
   ```
   horse-race-multiplayer.html 수정 필요:
   - countdown 시작: horse-race_countdown
   - 레이스 시작(0 직후): horse-race_gunshot
   - 레이스 중(루프): horse-race_crowd 또는 horse-race_bgm
   - 1등 골인: horse-race_finish
   - 결과 오버레이: horse-race_result
   ```

2. **빠른 테스트용 placeholder 활용**
   - 현재 8KB 무음 파일이 있으므로 실제 에셋만 교체하면 됨
   - `/addsound` 커맨드로 추가 시 자동 placeholder 생성

3. **룰렛 Web Audio API 연동**
   - 현재 `RouletteSound` 객체가 합성음 사용 중
   - mp3 에셋과 중복 재생 방지 필요

### 디자인팀

#### 2-5. UI/UX 디자이너 (C)

**UX 관점**

1. **사운드 피드백 타이밍 정밀화**
   - 버튼 클릭음: 즉시 (50ms 이내)
   - 결과 확정음: 애니메이션 완료 직후
   - 타이밍 어긋나면 오히려 UX 저해

2. **모바일 진동(Haptic) 연계 고려**
   - `navigator.vibrate()` API
   - 사운드 OFF 유저에게도 피드백 제공
   - 주사위 결과, 룰렛 당첨 시 짧은 진동

3. **사운드 설정 UI 개선**
   - 현재: 단순 ON/OFF 토글
   - 제안: 효과음/BGM 분리 토글 + 볼륨 슬라이더

#### 2-6. 사운드 디자이너 (D)

**사운드 선정 가이드라인**

1. **게임별 사운드 톤 제안**

   | 게임 | 톤/무드 | 키워드 |
   |------|---------|--------|
   | 주사위 | 캐주얼, 가벼움 | 나무 주사위, 탁자 소리 |
   | 룰렛 | 긴장감, 카지노 | 휠 돌아가는 소리, 볼 튀는 소리 |
   | 경마 | 역동적, 스포츠 | 말발굽, 함성, 팡파레 |
   | 공통 | 중립적, 깔끔 | UI 클릭, 알림 |

2. **효과음 소싱 추천 사이트**
   - **freesound.org**: CC0 다수, 검색 용이
   - **pixabay.com/sound-effects**: 상업 무료
   - **zapsplat.com**: 회원가입 필요, 고품질

3. **경마 사운드 구체적 추천**
   - countdown: "3, 2, 1" TTS 또는 비프음
   - gunshot: 클래식 출발 신호총
   - crowd: 경마장 환호성 (루프 가능한 것)
   - finish: 트럼펫 팡파레 또는 승리 징글
   - bgm: 스포츠 중계 느낌 BGM (120-140 BPM)

### 품질팀

#### 2-7. QA 엔지니어 (E)

**테스트 방법론**

1. **사운드 재생 검증 방법**
   - 브라우저 개발자 도구 Network 탭에서 mp3 요청 확인
   - `SoundManager.playSound()` 호출 시 콘솔 로그 추가 (디버그용)
   - 각 게임 이벤트별 사운드 트리거 체크리스트

2. **크로스 브라우저/디바이스 테스트**
   - Chrome, Safari, Firefox, Edge
   - iOS Safari (autoplay 제한 특이사항)
   - 안드로이드 Chrome

3. **에셋 교체 후 회귀 테스트**
   - placeholder → 실제 에셋 교체 시 파일명/경로 일치 확인
   - 파일 용량 변화로 인한 로딩 지연 테스트

### 관점 간 충돌 사항

1. **BGM 필요성** — 기획 사원(필요) vs 개발 팀장(선택적)
   - 결론: 경마만 우선 적용, 나머지는 추후 판단

2. **모바일 기본 사운드 설정** — 기획 사원(OFF 권장) vs 현재 구현(ON)
   - 결론: 당장은 현행 유지, 추후 UX 테스트 후 결정
