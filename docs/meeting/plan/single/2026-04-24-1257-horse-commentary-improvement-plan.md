# 회의록: 경마 중계 재미와 타이밍 개선
> 일시: 2026-04-24

## 안건

경마 중계 멘트가 식상하고, 실제 레이스 타이밍과 맞지 않는 문제가 있다. 사람 이름을 섞고, 최근 추가된 기믹 발동 시에도 중계가 반응하도록 개선 방향을 문서로 정리한다.

## 현재 문제

- 중계가 3초마다 랜덤 문장을 보여주는 구조라 실제 상황과 맞지 않는다.
- 선두 변경, 접전, 결승 직전, 기믹 발동 같은 중요한 순간이 일반 멘트에 묻힌다.
- 플레이어 이름과 선택 탈것이 거의 언급되지 않아 “내 경기” 느낌이 약하다.
- 진화/낙마/부스터 같은 최근 연출이 켜져도 해설이 알아채지 못한다.
- 글씨가 바뀌는 연출이 모두 같아서 중요한 멘트와 평상시 멘트의 무게가 구분되지 않는다.

## 개선 방향

### 1. 랜덤 중계에서 상황 중계로 전환

중계 문구를 단일 랜덤 배열이 아니라 상황별 풀로 나눈다.

| 상황 | 예시 |
|------|------|
| 출발 직후 | "초반 탐색전입니다. 아직은 다들 숨을 고릅니다." |
| 선두 유지 | "{이름}의 {탈것}, 페이스가 안정적입니다." |
| 선두 변경 | "{이름}의 {탈것} 선두 탈환! 순위표가 흔들립니다." |
| 접전 | "{선두}와 {추격자}, 간격 {거리}m. 거의 붙었습니다." |
| 결승 직전 | "남은 거리 {거리}m. 이제 실수 한 번이면 끝입니다." |
| 완주 | "{이름}의 {탈것} {순위}등 통과!" |

### 2. 사람 이름을 자연스럽게 넣기

선택한 사람이 1명이면 `{이름}의 자동차`, 2명이면 `{A}·{B}의 자동차`, 3명 이상이면 `{A} 외 2명의 자동차`처럼 표시한다.

이름은 모든 멘트에 넣지 않고, 중요한 순간에만 넣는 편이 좋다. 계속 이름을 반복하면 오히려 피로해진다.

### 3. 기믹 발동 중계 추가

기믹 발동 시 일반 랜덤 멘트를 잠깐 멈추고 이벤트 멘트를 우선 표시한다.

| 기믹 | 멘트 방향 |
|------|-----------|
| sprint | "{이름}의 {탈것} 가속! 뒤에서 바람이 터집니다." |
| stop | "{이름}의 {탈것} 급정거! 브레이크가 먼저 말했습니다." |
| slow | "{이름}의 {탈것}, 지금 구간은 버티는 게 목표입니다." |
| slip | "{이름}의 {탈것} 미끄러집니다! 중심을 다시 잡아야 합니다." |
| item_boost | "황금 당근 발동! 이건 간식이 아니라 추진력입니다." |
| item_trap | "바나나 함정! 방금 트랙이 웃었습니다." |
| reverse | "역주행! 지금 보는 게 맞습니다. 반대로 갑니다." |
| reverse_boost | "역주행 보상 부스트! 손해 본 만큼 돌려받습니다." |
| evolution_charge | "주변이 번쩍입니다. 각성 전조가 들어왔습니다." |
| evolution | "진화 발동! 지금부터 장르가 살짝 바뀝니다." |
| evolution_boost | "각성 질주 시작! 하이라이트 후보입니다." |
| fallen/finish-stun | "결승선 앞에서 넘어집니다! 아직 순서가 정리되지 않았습니다." |

### 4. 글씨 연출을 중요도별로 나누기

현재처럼 같은 상태바에 같은 방식으로만 바꾸면 멘트의 강약이 없다. 화면을 가리지 않는 선에서 상태를 나눈다.

| 등급 | 용도 | 연출 |
|------|------|------|
| live | 일반 진행 | 조용한 텍스트 교체 |
| event | 선두 변경/접전 | 짧은 팝 애니메이션 |
| gimmick | 기믹 발동 | 노란 강조, 2~3초 유지 |
| final | 결승 직전/완주 | 붉은 강조, 살짝 긴 유지 |

중요한 점은 새 UI를 크게 띄우기보다 기존 `gameStatus` 영역을 활용하는 것이다. 경주 화면을 가리면 관전성이 떨어진다.

### 5. 타이밍 규칙

- 일반 멘트는 4초 안팎으로 천천히 교체한다.
- 기믹 멘트는 발동 순간 즉시 표시하고 2.5~3.5초 유지한다.
- 선두 변경 멘트는 너무 자주 나오지 않게 2~3초 쿨다운을 둔다.
- 접전 멘트는 8초 이상 쿨다운을 둔다.
- 결승 100m, 50m, 15m 구간은 각각 한 번만 말한다.
- 다시보기에서는 과한 실시간 중계보다 기존 결과 재생 안정성을 우선한다.

## 구현 메모

코드 변경 시 추천 구조:

- `js/horse-race-commentary.js`
  - 상황별 멘트 풀
  - 기믹별 멘트 풀
  - 이름/탈것 표시 포맷터
  - `startRaceCommentary()`, `updateRaceCommentaryContext()`, `announceRaceMoment()`
- `js/horse-race.js`
  - 애니메이션 루프에서 선두/2등/거리/남은 거리만 전달
  - 기믹 트리거 지점에서 `announceRaceMoment('gimmick', ...)`
  - 결승 판정 시 `announceRaceMoment('finish', ...)`
- `css/horse-race.css`
  - `live/event/gimmick/final` 등급별 가벼운 텍스트 전환 연출

## 문구 톤 가이드

- 너무 스포츠 중계만 흉내 내기보다 LAMDice 특유의 가벼운 농담을 섞는다.
- 플레이어를 놀릴 때는 결과를 비난하지 말고 상황을 웃긴다.
- 기믹 멘트는 짧고 즉시 이해돼야 한다.
- 이름은 중요한 순간에만 넣는다.
- "운명", "각성", "하이라이트" 같은 단어는 남발하지 않는다.

## 우선순위

1. 상황별 중계 풀 분리
2. 플레이어 이름 + 탈것명 포맷 추가
3. 기믹 발동 멘트 추가
4. 중요도별 텍스트 연출 추가
5. 탈것별 캐릭터 해설 추가

## 결정

- 채택: 중계는 랜덤 교체 중심에서 상황 이벤트 중심으로 개선한다.
- 채택: 사람 이름은 중요한 순간에만 자연스럽게 넣는다.
- 채택: 진화, 부스터, 역주행, 바나나, 낙마/스턴 계열은 별도 멘트를 둔다.
- 보류: AI 실시간 해설 생성은 호출량과 타이밍 문제 때문에 이번 범위에서 제외한다.
- 보류: 큰 자막 오버레이는 화면을 가릴 수 있어 기존 상태바 개선 후 재검토한다.
