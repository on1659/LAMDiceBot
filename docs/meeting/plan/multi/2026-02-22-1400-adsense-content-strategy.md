# LAMDiceBot 팀 회의록

**일시**: 2026-02-22
**주제**: Google AdSense 심사 통과를 위한 콘텐츠 전략
**참석자**: 기획자, 개발자, QA/UX
**회의 방식**: 3인 멀티에이전트 독립 분석 → 교차 검토 → 합의 도출

---

## 1. 현황 요약

- Google AdSense "가치 없는 콘텐츠(Low-value content)"로 거절
- 메인 화면 리디자인 완료 (CTA, 태그라인 롤링, LAMDice 브랜딩)
- 기존 정보성 페이지: about-us, dice-rules-guide, probability-analysis, contact, privacy-policy, terms-of-service, statistics (총 7개)
- SEO 기본: OG 태그, canonical URL, meta description, sitemap.xml, robots.txt, ads.txt 존재
- 게임 페이지 3개에 JSON-LD 적용됨, 정보 페이지에는 미적용

## 2. 독립 분석 결과

### 2-1. 기획자 (A)

1. **게임 가이드 허브 페이지** (우선순위: 상) — 룰렛/경마 가이드 추가 + 전체 가이드 목록 페이지 생성. 콘텐츠 중심 사이트 전환의 핵심.
2. **게임 페이지 SEO 레이어** (우선순위: 상) — 각 게임 HTML에 noscript 규칙 텍스트 + 시맨틱 태그. 크롤러의 "빈 페이지" 인식 해결.
3. **사이트 전체 네비게이션 통일** (우선순위: 상) — 모든 정보 페이지에 동일한 헤더/푸터. "사이트 구조 불명확" 해결.
4. **공정성 검증 페이지** (우선순위: 중) — 난수 생성 원리 + 실 데이터 분포. 독창적 콘텐츠.
5. **statistics.html 정적 보강** (우선순위: 중) — noscript + 정적 설명 텍스트. 크롤러 빈 페이지 방지.

### 2-2. 개발자 (B)

1. **JSON-LD 구조화 데이터** (난이도: 하) — 정보 페이지에 Organization, Article, FAQPage 스키마. 리치 결과 노출.
2. **statistics.html SEO 강화** (난이도: 하~중) — 정적 텍스트 추가 또는 서버 사이드 HTML 생성.
3. **업데이트 로그 공개 페이지** (난이도: 하) — update-log.md → changelog.html. "살아있는 사이트" 신호.
4. **내부 링크 구조 강화** (난이도: 하) — 공통 네비게이션 + BreadcrumbList JSON-LD.
5. **sitemap.xml 개선** (난이도: 하) — lastmod, changefreq 추가.

### 2-3. QA/UX (C)

1. **메인 페이지 오버레이 문제** (리스크: 상) — z-index:10000 오버레이가 header/main/footer 전체 차단. 크롤러가 내부 링크를 발견 못함. 오버레이 하단에 콘텐츠 링크가 없음(privacy, terms, contact만).
2. **정보성 페이지 간 네비게이션 부재** (리스크: 상) — "메인으로 돌아가기" 외 상호 연결 없음.
3. **모바일 콘텐츠 접근성** (리스크: 중) — 오버레이 하단 링크 너무 작고 투명, padding 과도.
4. **광고 슬롯 배치 공간 부재** (리스크: 중) — Auto ads만, 수동 유닛 없음.
5. **FOUC opacity:0 문제** (리스크: 하) — 크롤러 빈 페이지 인식 가능성.

## 3. 교차 검토 결과

### 3-1. 기획자 검토
- B1(JSON-LD): 부분동의 — 네비게이션 문제 해결 후 효과 최대화
- B3(changelog): 동의 — "살아있는 사이트" 신호로 효과적
- B4(내부 링크): 동의 — C2와 연계, 높은 우선순위
- B5(sitemap): 동의 — 간단하고 실질적 효과
- C1(오버레이): 긴급 동의 — 크롤러 가시성 확보 필수
- C2(네비게이션): 동의 — 인덱싱 효율 + UX 동시 개선
- C4(광고 슬롯): 반대 — 현 트래픽 규모에서 과도한 투자

### 3-2. 개발자 검토
- A1(가이드 허브): 동의 — 순수 HTML, 간단
- A2(SEO 레이어): 동의 — 각 게임 HTML에 1-2시간
- A3(네비게이션): 부분동의 — EJS 없이 중복 감수, 최소 공통 헤더 추천
- A4(공정성 검증): 동의 — API 엔드포인트 추가 필요, 난이도 중
- C1(오버레이 구조 변경): 동의 — 난이도 높음, 링크 추가가 현실적
- C2(네비게이션): 동의 — 난이도 낮음, A3과 연계

### 3-3. QA/UX 검토
- A1(가이드 허브): 동의 — 사용자 진입 경로 단순화
- A2(SEO 레이어): 동의 — 크롤러 콘텐츠 수집 향상
- A4(공정성 검증): 반대 — about-us에 이미 충분, 중복 우려
- B1(JSON-LD): 동의 — 리치 결과 클릭률 향상
- B3(changelog): 동의 — 신뢰도 상승
- B4(내부 링크): 동의 — BreadcrumbList로 검색 경로 시각화
- B5(sitemap): 동의 — 인덱싱 속도 향상

### 의견 충돌 사항

- **공정성 검증 페이지**: A/B는 독창적 콘텐츠로 긍정, C는 about-us와 중복 우려로 반대 → 보류
- **오버레이 구조 변경 vs 링크 추가**: C는 구조 변경(난이도 높음), B는 링크 추가(현실적) → 링크 추가로 합의
