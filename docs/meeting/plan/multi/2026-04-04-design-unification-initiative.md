# 디자인 통합 이니셔티브 (Design Unification Initiative)

> **작성일**: 2026-04-04
> **상태**: 계획 수립 완료 → 1차 회의 대기
> **목표**: 4개 게임(주사위/룰렛/크레인/경마) 디자인·레이아웃 전체 통일

---

## 1. 배경

### 현재 상태
4개 게임이 독립적으로 개발되면서 레이아웃/디자인 일관성이 부족한 상태.

| 게임 | HTML 파일 | 용량 | CSS 접근 |
|------|----------|------|---------|
| 주사위 | dice-game-multiplayer.html | 365KB | theme.css + 인라인 |
| 룰렛 | roulette-game-multiplayer.html | 162KB | theme.css + 인라인 |
| 크레인 | crane-game-multiplayer.html | 130KB | theme.css + 인라인 |
| 경마 | horse-race-multiplayer.html | 56KB | theme.css + horse-race.css + Tailwind |

### 공통 인프라 (이미 존재)
- **css/theme.css** — CSS 변수 시스템 (색상 팔레트, 시맨틱 토큰, 게임별 색상)
- **8개 shared 모듈** — ControlBar, ReadyModule, ChatModule, OrderModule 등
- **다크모드 지원** — `[data-theme="dark"]` 변수 오버라이드

### 문제점
- 게임마다 레이아웃 구조가 다름 (로비, 게임 화면, 결과 화면)
- 인라인 스타일과 CSS 변수 혼재
- 경마만 Tailwind CSS 사용 (나머지는 순수 CSS)
- 게임별 반응형 브레이크포인트 불일치
- 공통 컴포넌트(채팅, 랭킹, 준비시스템) 배치 위치 불통일

---

## 2. 목표

### 통일할 대상
1. **레이아웃 구조** — 로비/게임/결과 화면의 영역 배치
2. **컴포넌트 배치** — 컨트롤바, 채팅, 랭킹, 준비시스템 위치 통일
3. **CSS 시스템** — 인라인 스타일 제거, CSS 변수 100% 전환
4. **반응형 전략** — 공통 브레이크포인트 및 모바일 레이아웃
5. **상태 표현** — 대기/진행/결과 화면 전환 패턴 통일

### 통일하지 않을 대상 (게임 고유 정체성)
- 게임별 테마 색상 (주사위=보라, 룰렛=빨강, 경마=갈색, 크레인=파랑)
- 게임 보드 자체의 디자인 (주사위판, 룰렛 휠, 트랙, 크레인기)
- 게임 고유 사운드/애니메이션

## 3. 회의 계획 — 2단계 분리 회의

### 왜 2단계로 나누는가
프로그래머가 있을 때와 없을 때 회의의 성격이 달라진다:
- **1차 (프로그래머 제외)**: 이상적인 디자인 방향을 자유롭게 논의. "할 수 있냐"가 아닌 "어떻게 해야 하냐"에 집중
- **2차 (프로그래머 포함)**: 1차 결정사항의 기술적 실현 가능성 검증 및 조율. 1차 결과가 무조건 확정은 아님

### 1차 회의: 기획/UI/PD

| 항목 | 내용 |
|------|------|
| **참석자** | 지민(PD), 현우(기획①), 소연(기획②), 다은(UI), 승호(UX) |
| **제외** | 태준(백엔드), 미래(프론트엔드), 윤서(QA) |
| **목표** | 디자인 통합 방향 결정, 레이아웃 표준안 도출, UI 가이드라인 초안 |
| **실행 방법** | `/meeting-team` — 5명 지정 참여 |

**1차 회의 안건:**
1. 현재 4개 게임 레이아웃 분석 및 문제점 공유
2. 통일 레이아웃 표준안 제안 (로비/게임/결과)
3. 공통 컴포넌트 배치 규칙 (채팅, 랭킹, 준비시스템)
4. 반응형 전략 통일 (모바일/데스크톱)
5. CSS 시스템 정리 방향 (인라인 제거, 변수화)
6. 경마 Tailwind 의존성 처리 방안

### 2차 회의: 전체 팀 (프로그래머 합류)

| 항목 | 내용 |
|------|------|
| **참석자** | 전원 (8명) |
| **추가 참석** | 태준(백엔드), 미래(프론트엔드), 윤서(QA) |
| **목표** | 1차 결정사항 기술 검증, 구현 일정 수립, 리스크 도출 |
| **실행 방법** | `/meeting-team` — 전원 참여 |

**2차 회의 안건:**
1. 1차 회의 결과 공유 및 기술적 피드백
2. 구현 가능성 조율 (1차 결정 수정/보완)
3. 구현 순서 및 공수 산정
4. 공통 모듈 수정 범위 (shared-*.js 영향)
5. 게임별 마이그레이션 전략 (한 번에 vs 순차)
6. QA 전략 — 크로스게임 검증 계획

## 4. 지속 팀원 시스템 (Persistent Agent Team)

### 개념
회의에 참여하는 에이전트들은 1회성이 아니라 프로젝트와 함께 지속되는 팀원.
회의 결과, 결정사항, 각 팀원의 의견 이력을 메모리에 저장하여 다음 세션에서도 같은 맥락으로 이어간다.

### 저장 위치
```
~/.claude/projects/d--Work-LAMDiceBot/memory/
├── MEMORY.md                          # 인덱스
├── team_design_meeting_phase1.md      # 1차 회의 결과 (작성 예정)
├── team_design_meeting_phase2.md      # 2차 회의 결과 (작성 예정)
├── team_design_decisions.md           # 확정된 디자인 결정사항 (작성 예정)
└── ...
```

### 저장 내용
- 각 팀원이 제시한 의견 및 반대 의견
- 합의된 결정사항과 그 근거
- 미결 이슈 및 후속 논의 필요 사항
- 각 팀원의 전문 판단 이력 (다음 회의에서 참조)

## 5. 텔레그램 보고 시스템 (후순위)

### 현재 상태
- 봇 생성 완료 (BotFather)
- 토큰: `.env`에 `TELEGRAM_BOT_TOKEN`으로 저장 (gitignore 처리됨)

### 계획
- 보고 시점: 회의 완료, 구현 결과, 주요 의사결정
- AI 모델: ZAI API GLM5 사용하여 요약 생성 → 텔레그램 전송
- 세팅 페이지: `/botpage` 등 전용 관리 페이지 구축
- **구현 시기**: 디자인 통합 회의 및 초기 구현 이후

### 보고 포맷 (예정)
```
📋 [회의 완료] 디자인 통합 1차 회의
━━━━━━━━━━━━━━━━━━
참석: 지민, 현우, 소연, 다은, 승호
결정사항: 3건 / 미결: 1건
핵심: 공통 레이아웃 3단 구조 채택
━━━━━━━━━━━━━━━━━━
상세: docs/meeting/plan/multi/2026-04-04-...
```

## 6. 실행 순서

```
[현재] 문서 작성 및 정리 ✅
  ↓
[다음] 1차 디자인 회의 (PD/기획/UI/UX — 프로그래머 제외)
  ↓ 결과 저장 → 메모리 파일
  ↓
[이후] 2차 전체 회의 (프로그래머 합류 — 기술 검증 및 조율)
  ↓ 결과 저장 → 메모리 파일
  ↓
[확정] 디자인 통합 구현 계획서 작성
  ↓
[구현] 게임별 순차 마이그레이션
  ↓
[QA] 크로스게임 검증
  ↓
[후순위] 텔레그램 보고 시스템 구축
```

## 7. 참고: 현재 게임별 레이아웃 구조 비교

### 공통 패턴 (모든 게임)
```
body → div.container
  ├── section.lobby-section (로비)
  └── section.game-section (게임, display:none)
       ├── ControlBar (shared)
       ├── [게임 보드 영역]
       ├── ChatModule (shared)
       └── ReadyModule (shared)
```

### 게임별 차이점
| 영역 | 주사위 | 룰렛 | 크레인 | 경마 |
|------|--------|------|--------|------|
| CSS 방식 | theme.css + 인라인 | theme.css + 인라인 | theme.css + 인라인 | theme.css + horse-race.css + Tailwind |
| 게임보드 구조 | 복잡 (배팅+주사위+결과) | 중간 (휠+배팅) | 간단 (크레인+아이템) | 복잡 (트랙+차량+순위) |
| 채팅 위치 | 게임 보드 하단 | 게임 보드 하단 | 사이드 | 사이드 |
| 랭킹 표시 | 인라인 | 인라인 | 없음 | 별도 패널 |
| 히스토리 | 좌측 사이드 | 좌측 사이드 | 없음 | 좌측 사이드 |

> 이 비교표는 1차 회의의 기초 자료로 사용됩니다.
