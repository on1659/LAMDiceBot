# 디자인 통합 회의 결과 (최종)

> **날짜**: 2026-04-04 ~ 04-05
> **참석**: 지민(PD), 현우(기획①), 소연(기획②), 다은(UI), 승호(UX), 태준(백엔드), 미래(프론트), 윤서(QA)
> **상태**: 확정 — Go
> **대상**: 주사위, 룰렛, 경마 (크레인 완전 제외)

---

## 최종 결정사항

### 1. 대상 게임

**주사위, 룰렛, 경마** 3개만. 크레인은 비공개 상태이며 완전 제외.

### 2. 핵심 원칙

- **실제 보이는 화면** 통일이 목표 — CSS 리팩토링이 목적이 아님
- 로비는 **하나** — `server-select-shared.js` 공통 서버 선택 UI를 각 게임이 공유
- 게임 중 광고 **허용** — 단, 레이아웃이 깨지지 않는 배치 (경마 현재 레이아웃 참고)
- **같은 게임 사이트라는 느낌** 유지 — 색상이 완전 동일할 필요는 없지만 톤 통일
- Phase별 순차 적용

---

## 눈에 보이는 통일 항목

### A. 색상 톤 통일 (최우선)

**문제**: 보라(주사위) / 회색(룰렛) / 갈색(경마) — 같은 사이트 느낌이 전혀 안 남.

**방향**: 각 게임 고유색은 유지하되, **채도/명도 톤을 맞춰서** 하나의 브랜드로 느껴지게.

| 게임 | 현재 | 문제 | 변경 방향 |
|------|------|------|----------|
| 주사위 | 보라 `#667eea→#764ba2` | 채도 높고 깔끔 | 기준점 (이 톤에 맞춤) |
| 룰렛 | 회색 `#8e99a4→#6b7c8a` | 정체성 없음, 촌스러움 | 딥그린+골드 (`#1a5c3a→#2d7a52`, accent `#c9a84c`) |
| 경마 | 갈색 `#8B4513` | 너무 투박, 보라와 톤 안 맞음 | 톤 조정 필요 (채도/명도를 주사위와 맞춤) |

공통 요소(버튼, 패널, 텍스트)의 색상은 `theme.css` 공유 토큰 사용으로 통일.

### B. 랭킹 통일 (긴급)

**문제**: 게임마다 랭킹 UI가 완전히 다름.

| 게임 | 현재 랭킹 | 문제 |
|------|----------|------|
| 주사위 | `ranking-shared.js` 오버레이 (풀스크린) | 기준 |
| 룰렛 | `ranking-shared.js` 오버레이 (풀스크린) | 주사위와 동일, OK |
| 경마 | `liveRankingPanel` 인라인 하드코딩 (게임 내 인라인 패널) | **완전 이질적** — 별도 스타일, 별도 구조 |

**방향**: 경마 랭킹을 `ranking-shared.js` 오버레이 방식으로 통일. 또는 역으로 인게임 인라인 패널을 3개 게임 공통 모듈로 만들기.

### C. 광고 배치 (레이아웃 유지 조건)

**원칙**: 게임 중 광고 허용. 단 레이아웃이 깨지지 않아야 함.

| 위치 | 판단 | 상세 |
|------|------|------|
| 로비 | 유지 | 방 목록 아래, 3게임 동일 위치 |
| **게임 중** | **허용** | 경마처럼 게임 영역 밖(사이드 or 하단)에 배치. 게임 보드를 침범하지 않는 위치 |
| 결과 화면 | 추가 | 결과 표시 후 하단에 1개 |
| 대기 시간 | 추가 (선택) | 호스트 시작 전 빈 공간 |
| 페이지 하단 | 수정 | SEO 섹션에 묻힌 현재 위치 → 실제 보이는 하단으로 |

**참고 모델**: 경마 현재 레이아웃 — 게임 영역 + 사이드 채팅이 잘 분리되어 있어 광고 배치 공간 확보 가능.

### D. 히스토리 패널 동작 통일

- 3개 모두 좌측 사이드이나 열리는 방식/헤더 다름
- 열기/닫기 방식, 타이틀 스타일 통일

### E. 상태 전이 피드백 통일

- 대기→준비→진행→결과 시각적 표현을 3개 게임 통일
- 상태 바, 카운트다운, 결과 연출 패턴

## 타임라인

| 단계 | 내용 | 기간 |
|------|------|------|
| Phase 1 | 색상 톤 통일 (룰렛 변경 + 경마 톤 조정 + 공통 토큰) | 1일 |
| Phase 2 | 랭킹 UI 통일 (경마 liveRankingPanel → 공통 방식) | 1~2일 |
| Phase 3 | 게임 중 광고 배치 (레이아웃 유지 조건) | 1일 |
| Phase 4 | 로비 광고 위치 통일 + 하단 광고 수정 | 0.5일 |
| Phase 5 | 히스토리 패널 + 상태 전이 통일 | 1일 |
| QA | 3게임 크로스 검증 | 1일 |
| **총** | | **약 5~6일** |

## 기술 검증 요약

- **백엔드 영향 없음** — socket/DB 변경 불필요 (태준)
- **shared 모듈 cssText 건드리지 말 것** — 4게임 동시 영향 (태준)
- **QA Gate 순차 검증** — Gate 통과 못하면 다음 진행 불가 (윤서)
- **게임별 feature 브랜치 격리** (태준)
- **로비는 공통** — `server-select-shared.js`가 3게임 공유
