> **날짜**: 2026-04-04
> **1차 참석**: 지민(PD), 현우(기획①), 소연(기획②), 다은(UI), 승호(UX)
> **2차 참석**: 전원 + 태준(백엔드), 미래(프론트엔드), 윤서(QA)
> **상태**: 확정 — Go

## 최종 결정사항

### 1. 적용 순서
크레인 → 경마 → 룰렛 → 주사위 (작은 것부터, 주사위 마지막)

### 2. Tailwind 제거
확정. 현재 거의 미사용 상태. CDN 의존성 제거로 로딩 속도 개선.

### 3. 채팅 위치 — 하단 고정
- 모바일 비율 높음, 우측 고정은 모바일에서 깨짐
- 데스크톱(≥768px): 하단 채팅 유지
- 모바일(<768px): 접기/펼치기 토글 버튼
- 향후 데스크톱 전용 우측 패널은 별도 Phase에서 재논의 가능

### 4. 인라인 처리 — 별도 Phase
- Phase 1: layout.css + 쉘/슬롯 구조 통일
- Phase 2: 인라인 스타일 제거 (Phase 1 완료 후)
- 혼합 금지 — 추적 불가 리스크

### 5. 반응형 브레이크포인트 — 2개로 통일
- ≥768px: 데스크톱
- <768px: 모바일

### 6. 게임별 색상 정체성
| 게임 | 색상 | CSS 변수 |
|------|------|---------|
| 주사위 | 보라 | `--game-accent: #7c3aed` |
| 룰렛 | 빨강 | `--game-accent: #dc2626` |
| 크레인 | 파랑 | `--game-accent: #2563eb` |
| 경마 | 초록 | `--game-accent: #16a34a` |

### 7. CSS 파일 구조
```
css/
├── theme.css        → 기존 유지 (전역 앱 테마)
├── layout.css       → 신규: 쉘+슬롯 공통 레이아웃
├── dice.css         → 신규: 주사위 전용
├── roulette.css     → 신규: 룰렛 전용
├── crane.css        → 신규: 크레인 전용
└── horse-race.css   → 기존: 경마 전용 (레이아웃 분리)
```

## 기술 검증 결과 (2차 회의)

### 태준 (백엔드)
- 백엔드 영향 **없음** (socket/DB 변경 불필요)
- layout.css는 정적 파일 추가만으로 서버 재시작 불필요
- shared 모듈 내 cssText **건드리지 말 것** (4게임 동시 영향)
- 게임별 feature 브랜치 격리 권장

### 미래 (프론트엔드)
- JS `.style.*` 조작: dice 312건 중 CSS 이동 가능 후보 30~40건 (90%는 그대로)
- `display toggle` (40%), `animation` (15%), `cssText 덩어리` (22%) — 대부분 CSS 외부화와 무관
- layout.css 범위: 쉘 구조 4존으로 제한 (다이얼로그/오버레이 제외)
- 모바일 채팅: 접기/펼치기 토글 제안 (게임 보드 100% 활용)
- 총 프론트 공수: 약 9일

### 윤서 (QA)
- 4 Gate 순차 검증 (크레인→경마→룰렛→주사위)
- Gate 통과 못하면 다음 게임 진행 불가
- 주요 리스크: CSS 글로벌 오염(상), JS 인라인 충돌(상), z-index 레이어(중)
- 총 QA 공수: 약 6일
- main 머지 전: 자동 테스트 + 6시나리오 수동 QA + 멀티플레이어 동기화 필수

### 지민 (PD) 최종 판단
- **GO** 확정
- 선행 조건: layout.css 구현 스펙 1페이지 문서화
- Phase 1 착수 전 스펙 합의 필수
