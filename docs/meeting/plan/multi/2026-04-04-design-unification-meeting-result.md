# 디자인 통합 회의 결과 (1차 + 2차 + 재회의)

> **날짜**: 2026-04-04
> **1차 참석**: 지민(PD), 현우(기획①), 소연(기획②), 다은(UI), 승호(UX)
> **2차 참석**: 전원 + 태준(백엔드), 미래(프론트엔드), 윤서(QA)
> **재회의**: 전원 5명 (실제 보이는 UI + 광고 배치 집중)
> **상태**: 확정 — Go
> **대상**: 주사위, 룰렛, 경마 (크레인 완전 제외)

---

## 최종 결정사항

### 1. 대상 게임
**주사위, 룰렛, 경마** 3개만. 크레인은 비공개 상태이며 이 프로젝트에서 완전 제외.

### 2. 적용 순서
경마 → 룰렛 → 주사위 (작은 것부터, 주사위 마지막)

### 3. 핵심 원칙
- **실제 보이는 화면** 통일이 목표 — CSS 리팩토링이 목적이 아님
- 게임 중 광고 **금지** (멀티플레이어 동기화 + UX 보호)
- Phase별 순차 적용 — 한번에 몰아서 하지 않는다
- 각 게임 고유 정체성(색상, 게임보드) 유지

---

## 눈에 보이는 통일 항목

### A. 로비 화면 통일
- 방 목록, 방 생성 버튼, 광고 위치를 3개 게임 동일하게
- 광고(ad-lobby)는 방 목록 아래, max-width 800px

### B. 채팅 위치
- 주사위/룰렛: 하단 (현 상태 유지)
- 경마: 사이드 (현 상태 유지 — 트랙이 화면 전체 차지하므로)
- 통일 불가 사유 인정 — 게임 특성상 예외

### C. 히스토리 패널 동작 통일
- 3개 모두 좌측 사이드이나 열리는 방식/헤더가 다름
- 열기/닫기 방식, 타이틀 스타일 통일

### D. 룰렛 색상 정체성 확립
현재 룰렛은 회색 그라디언트로 정체성 없음. 변경 확정:

| 게임 | 현재 | 변경 |
|------|------|------|
| 주사위 | 보라 `#667eea→#764ba2` | 유지 |
| 룰렛 | 회색 `#8e99a4→#6b7c8a` | **딥그린+골드** `#1a5c3a→#2d7a52`, accent `#c9a84c` |
| 경마 | 갈색 `#8B4513` | 유지 |

### E. 상태 전이 피드백
- 대기→준비→진행→결과 상태 표시를 3개 게임 통일
- 상태 바 색상 전환, 카운트다운 등

## 광고 배치 전략

### 현재 상태
- 3개 게임 모두: 로비 배너 1개 + 페이지 하단 1개
- 게임 중: 없음
- 하단 광고: SEO 섹션(left:-9999px) 바로 위라 실제 노출 안 됨

### 확정 변경

| 위치 | 판단 | 상세 |
|------|------|------|
| 로비 배너 | 유지 | 방 목록 아래, 위치만 3게임 통일 |
| **결과 화면** | **신규 추가** | 오버레이 닫힌 후 게임 영역 하단에 1개. 라운드마다 노출 → 기존 대비 3~5배 임프레션 증가 |
| **대기 시간** | **신규 추가 (선택)** | 호스트 시작 전 빈 공간에 배너. gameState=waiting일 때만 표시 |
| 게임 진행 중 | **금지** | 애니메이션 성능, 멀티플레이어 동기화, UX 모두 불가 |
| 결과 오버레이 안 | **금지** | "확인" 버튼 전환율 저하 |
| 페이지 하단 | 위치 수정 | SEO 섹션에 묻힌 현재 위치 → 실제 보이는 하단으로 이동 |

### 광고 주의사항
- 동일 슬롯이 동시 2곳 렌더링 금지 (AdSense 정책)
- 결과 광고는 신규 슬롯 ID 필요할 수 있음
- 순차 적용 시 AdSense 학습 리셋 2~4주 — 한 게임씩

## 기술 검증 요약 (2차 회의)

- **백엔드 영향 없음** — socket/DB 변경 불필요 (태준)
- **JS .style.* 312건(dice) 중 CSS 이동 가능 30~40건** — 90%는 그대로 (미래)
- **shared 모듈 cssText 건드리지 말 것** — 4게임 동시 영향 (태준)
- **QA 4Gate 순차 검증** — Gate 통과 못하면 다음 진행 불가 (윤서)
- **게임별 feature 브랜치 격리** (태준)
