## 1. v1 실패 진단

v1: 1440 × 320 (4.5:1) PNG로 받음. 실제 게임에서 컨텐츠가 너무 압축되어 디테일이 안 보임.

### 실측 원인

게임 코드 분석 (`js/horse-race.js:346`):
```js
const wallHeight = 6;
const availableTrackHeight = container.offsetHeight - 50;  // ≈ 350
const laneHeight = Math.min(75, Math.floor((availableTrackHeight - wallHeight*(N-1)) / N));
```

→ **lane height가 참가자 수에 따라 가변:**

| 참가자 수 | lane height |
|----------|-------------|
| 4명 | **75px** (max cap) |
| 5명 | 64px |
| 6명 | 53px |
| 7명 | 44px |
| 8명 | 38px |
| 9명 | 33px |
| 10명 | **29px** (min) |

### CSS 적용

`background-size: auto 100%` + `background-repeat: repeat-x`
→ 이미지 높이가 lane height에 강제 축소, 가로 비율 유지.

v1 1440×320 적용 시:

- 차량 sprite 세로: **60×45 중 lane 60px의 75%(45px)를 차지**
- 차량 sprite 위쪽 7.5px + 아래쪽 7.5px이 빈 공간 (vertical center로 인한 padding)

### 배경 컨텐츠 권장 배치 영역

캔버스 90px를 3구역으로:

```
┌──────────────────────────────┐  0px
│  상단 50% (45px) — 하늘/원경  │
│                              │
├──────────────────────────────┤  45px
│  중단 35% (32px) — 메인 모티프 │
│  ← 차량 sprite와 겹침 영역   │
├──────────────────────────────┤  77px
│  하단 15% (13px) — 지면      │
└──────────────────────────────┘  90px
```

**중단(차량 겹침)에 강한 색 또는 큰 모티프 배치 시 차량 가독성 박살.**

→ 메인 모티프는 **상단 50% + 중단 위쪽 15%**에 배치 (lane 위쪽 60% 영역). 차량은 vertical center라 lane 중앙 차지하지만, 차량보다 위에 모티프 두면 가시.

## 3.5. 스타일 가이드 (필독)

### 톤 — 모던 모바일 캐주얼 게임

**Reference 핵심: Crossy Road / Helix Jump / Subway Surfers / Sky Roller**

- **NO**: 어린이 그림책 도형(삼각 산 + 원 태양 + 사각 빌딩만)
- **NO**: 단순 2색 그라데이션 하늘 + 빈 단색 면 (허전, "찐따" 느낌의 주범)
- **NO**: 두꺼운 검은 외곽선 (cartoon toon은 OK지만 1~2px 얇은 색선만)
- **NO**: 모티프 1개 + 텅 빈 배경 (메인 모티프만 덩그러니)
- **NO**: 평면 단일 레이어 (모든 게 같은 깊이감)

### 시각 요소 (필수)

1. **레이어드 깊이감 (parallax-ready)**
   - 원경 (먼 산/구름/별) — 채도 낮고 흐림
   - 중경 (메인 모티프) — 채도 중·고, 디테일 풍부
   - 근경 (지면/풀) — 채도 높고 선명, 패턴 있음
   - → 3개 깊이 레이어로 공간감 표현

2. **Soft shadow / Subtle highlight**
   - 모티프마다 부드러운 음영 (drop shadow 5~10% 불투명 검정)
   - 빛 방향 일정 (예: 좌상단 → 우하단)
   - 하이라이트는 별도 색이 아니라 살짝 밝은 톤

3. **풍부한 색 팔레트**
   - 한 배경에 **5~7색** 사용 (3색 미만 금지)
   - 그라데이션은 2색이 아니라 **3색 이상** (예: 하늘 — 보라→분홍→오렌지)
   - 차량과 보색 강조는 유지하되, 보조 색 2~3개 더

4. **분위기 채움 (atmosphere fill)**
   - 빈 영역에 작은 입자/디테일 살포:
     - 우주 → 별 + 작은 행성/혜성
     - 노을 → 작은 구름 + 새 실루엣 (작게)
     - 화산 → 재 파티클 + 용암 튀김
     - 꽃밭 → 꽃잎 날림 + 작은 풀잎
   - "빈 공간 = 자연 풍경의 일부"가 되도록

5. **현대적 일러스트 디테일**
   - 그라데이션 + 음영 + 텍스처 살짝
   - flat 2D인데도 dimensional하게 보이는 톤
   - 게임 UI 아이콘 디자인 트렌드 (2024-2026 모바일 캐주얼 게임 톤)

### 색감 톤 (모든 배경 공통)

- 채도: 중~높음 (펄럭이는 느낌, 흐릿한 파스텔만은 NO)
- 명도: 모티프 명확 + 배경 적절한 콘트라스트
- 따뜻한 톤 + 차가운 톤 혼합 (단일 톤 = 단조로움)

### 비유

> 게임 차량(자전거, 토끼, 닌자)이 **무대 주인공**이면 배경은 **잘 만든 무대 세트**.
> 무대가 텅 빈 도화지면 어색하고, 너무 빽빽하면 차량이 묻힘.
> "이 차량이 이 풍경 속을 달린다"가 한눈에 보여야 함.

## 4. 컨텐츠 디자인 원칙

### 핵심 — 풍성한 풍경 (v1 의도 유지)

**"나무 하나 + 건물 하나"처럼 휑하게 만들지 말 것.** 가로 960px 전체가 하나의
완성된 풍경이어야 한다. v1에서 풍성하게 채운 방향은 옳았다 — 문제는 사이즈/비율뿐이었다.

| 항목 | v1 | v2 |
|------|-----|-----|
| 사이즈 | 1440×320 (4.5:1, 비율 안 맞음) | **960×90 (10.67:1, lane 매칭)** |
| 풍경 밀도 | **풍성함 (유지)** | **풍성함 (유지)** |
| 요소 구성 | 작은 디테일만 다수 | **큰 핵심 요소 + 중간 + 작은 디테일 — 크기 계층** |
| 외곽선 | 1px | **2~3px** |
| 스타일 | 평면적 | **모던 모바일 게임 톤 (3.5장)** |

### 풍경 밀도 — 크기 계층으로 풍성하게

한 배경 = 가로 960px를 가득 채우는 풍경. 다음 3계층을 모두 배치:

1. **핵심 요소 (40~70px, 1~2개)** — 그 배경의 정체성 (화산/등대/큰 나무/달).
   lane 29px에서도 식별 가능해야 함.
2. **중간 요소 (20~40px, 3~5개)** — 풍경을 채우는 반복 요소
   (나무 여러 그루, 빌딩 여러 채, 언덕 능선, 파도 등). 가로로 리듬감 있게 분포.
3. **작은 디테일 (5~20px, 다수)** — 분위기 입자
   (별, 꽃잎, 풀, 구름 조각, 새 실루엣). lane 작아지면 뭉개져도 "질감"으로 보이면 OK.

→ 핵심+중간+작은 요소가 **함께 가로 전체를 채워** 휑한 느낌이 없어야 한다.

### 가시성 픽셀 계산

캔버스 90px → 실제 lane 29px (최악) 시 0.32배 축소.

| 캔버스 크기 | lane 29px 환산 | 역할 |
|------------|---------------|------|
| 5~20px 디테일 | 1.6~6.4px | 작아지면 뭉개짐 — **질감/분위기용** (OK) |
| 20~40px 중간 | 6.4~13px | 흐릿하지만 형태 인지 — **풍경 채움** |
| **40px+ 핵심** | **13px+** | 명확히 식별 — **배경 정체성** |

→ 핵심 요소만 40px+ 보장. 중간·작은 요소는 풍성함을 위해 자유롭게.

### seamless 좌우 이음

캔버스 가로 960px가 800px lane에 거의 1회(75px lane 기준) → **이음이 거의 안 보임**.

lane height 작아지면 반복 빈도 늘어남. seamless 처리:

1. **하늘/그라데이션**: 좌우 끝 색 동일 (가로 그라데이션 금지, 위→아래만)
2. **지면 띠**: 좌우 끝 같은 톤/패턴, 가로로 연속
3. **중간/작은 요소**: 가로 전체에 리듬감 있게 분포 (나무 5그루를 균등하게 등) —
   좌우 끝에서 잘리는 요소는 반대편에서 이어지게 (seamless tiling)
4. **핵심 요소**: 좌우 끝(x=0, x=960)에 걸치지 않게 — 중앙 영역(x=200~760)에 배치

## 6. 차량별 컨셉 (정밀)

각 배경은 **핵심 요소(40px+) + 중간 요소 3~5개 + 작은 디테일 다수**로 가로 전체를
가득 채운 풍경. 아래는 핵심/중간/디테일 예시 — 휑하지 않게 풍성하게.

### 도로 계열
- **car** (자동차): 핵심=빌딩 실루엣 2~3채(40~70px, 높낮이 다르게) / 중간=가로수·가로등 여러 개 / 디테일=창문 불빛·구름. 하늘 3색 그라데 #87CEEB→#B0C4DE→#FFE5B4.
- **bicycle** (자전거): 핵심=큰 나무 1~2그루(50~65px) / 중간=작은 나무 3~4그루·잔디 언덕 능선 / 디테일=풀잎·꽃·구름. 하늘 #A8D8E8→#C8E6A0.
- **scooter** (스쿠터): 핵심=신호등+표지판(50px) / 중간=가로수·가로등·낮은 담장 여러 개 / 디테일=보도블록 패턴·구름. 회색 톤 #BDC3C7 + 포인트 색.
- **knight** (기사): 핵심=성벽+큰 빨간 깃발(60~70px) / 중간=망루·작은 깃발 여러 개 / 디테일=돌담 텍스처·구름. 어두운 회색 #5D6D7E.

### 하늘 계열
- **bird** (새): 핵심=노을 태양(55px 원) / 중간=구름 4~5개(크기 다르게) / 디테일=먼 새 실루엣·노을 빛결. 노을 3색 #FF9966→#F77F00→#C44536.
- **eagle** (독수리): 핵심=산봉우리 2~3개(50~75px 능선) / 중간=구름 3~4개 / 디테일=먼 봉우리·바람결. 하늘 #5DADE2→#BDE0FE.
- **helicopter** (헬기): 핵심=빌딩 2~3채(60~80px) / 중간=낮은 건물·구름 여러 개 / 디테일=옥상 헬리패드·창문. 하늘 #87CEEB→#FFFFFF.
- **rocket** (로켓): 핵심=행성 1~2개(40~60px 원) / 중간=작은 행성·혜성·성운 / 디테일=별 다수(크기 다르게)·우주먼지. 우주 #0A0E27→#1A1A40.

### 물 계열
- **boat** (보트): 핵심=등대(70~80px) / 중간=먼 섬·돛단배 실루엣·바위 / 디테일=파도 거품·갈매기·구름. 하늘 #87CEEB, 바다 3색 #2980B9→#5DADE2.
- **turtle** (거북): 핵심=야자수 1~2그루(60~75px) / 중간=작은 야자수·바위·먼 섬 / 디테일=파도 곡선·조개·구름. 하늘 #FFE5B4, 바다 #5DADE2.
- **crab** (게): 핵심=모래 언덕+큰 조개(45~50px) / 중간=작은 조개·불가사리·바위 여러 개 / 디테일=모래 알갱이 질감·파도. 모래 #FFD580 + 바다.

### 자연 계열
- **rabbit** (토끼): 핵심=큰 꽃 2~3송이(40~55px) / 중간=작은 꽃·클로버·버섯 여러 개 / 디테일=꽃잎 날림·풀잎. 진한 풀밭 #4F8A4F.
- **horse** (말): 핵심=언덕 능선 2~3겹(50~80px) / 중간=나무·울타리·건초더미 / 디테일=풀잎·들꽃·구름. 황금 들판 #DAA520→#9ACD32.
- **dinosaur** (공룡): 핵심=화산 1~2개(60~80px) / 중간=고사리·바위·작은 화산 / 디테일=재 파티클·용암 튀김·연기. 주황 하늘 #FF7F50→#8B2500.
- **ninja** (닌자): 핵심=보름달(55px 원) / 중간=산 실루엣 능선·먼 탑·대나무 / 디테일=별 다수·반딧불·안개. 짙은 남보라 #1A1A40.

> 위 "핵심/중간/디테일"은 가이드. **요점은: 가로 960px가 텅 비지 않고 완성된 풍경으로 가득 차야 함.**
> "나무 하나 + 건물 하나" 같은 휑한 구성 금지.

## 7. 검증 기준 (객관적)

생성된 PNG는 다음 자동 검증 통과해야 함:

1. **해상도 정확**: 정확히 960×90 (오차 0)
2. **파일 크기**: ~10KB 이하 (PNG-8 또는 적절히 최적화)
3. **seamless 검증**: x=0열 픽셀과 x=959열 픽셀의 RGB 평균 차이 ≤ 5
4. **차량 가독성 대비**: 차량 색상(매트릭스 참조)과 배경 중단(y=45~77) 평균 색의 luminance 비율 ≥ 3:1
5. **dominant color**: 컨셉에 명시된 톤과 일치 (육안 검증)

LAMDiceBot 측 픽업 후 검증:
- `AutoTest/background-readability-preview.html` 14종 동시 시각 검증
- `/horse-race` 실제 게임에서 4명/8명 케이스 모두 봇으로 진입시켜 가독성 확인
