# update-log 작성 규칙

- **수정할 파일**: `update-log.md` 만 사용. `update-log_old.md` 는 읽거나 수정하지 않음.
- **표시**: 게임 로비에서는 `update-log.md` + `update-log_old.md` 를 합쳐서 보여줌.

## 포맷 (Markdown)

- 날짜 블록: `## 🎮 최신 업데이트 (YYYY-MM-DD)` 또는 `## 🎮 이전 업데이트 (YYYY-MM-DD)`
- 항목 제목: `**이모지 제목**` (예: `**🐎 경마 안정성 개선**`)
- 본문: `- ` 로 시작하는 불릿 한 줄 이상
- 구분선: `---`

## 예시

```markdown
## 🎮 최신 업데이트 (2026-02-15)

**🐎 경마 안정성 개선**
- 경마 게임 접속 안정성을 개선했습니다.

**🔧 멤버 관리 버그 수정**
- 서버 멤버 버튼이 반응하지 않던 문제를 수정했습니다.

---

## 🎮 이전 업데이트 (2026-02-14)
...
```

## 포함/제외

- **포함**: 새 기능, 버그 수정, UI/UX 개선, 게임 규칙 변경 (플레이어 관점)
- **제외**: 리팩터링, 내부 로직, 봇/테스트/설정
