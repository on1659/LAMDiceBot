# 사다리타기 — 공개 연출 개선 + UX 다듬기 묶음 — Goal

사다리타기(ladder)의 공개(추적) 연출과 몇 가지 UX 문제를 한 번에 개선해라. feature/goal-game-mode 브랜치에서 작업한다.

> 전제: 직전까지 완료된 "유저 빌드 맵" + "레인 선택을 빌드 단계로 통합"이 적용된 상태 위에 얹는다.
> baseline 파일: `socket/ladder.js`, `js/ladder.js`, `ladder-multiplayer.html`, `css/ladder.css`, `tests/test-ladder.js`.

## 한 줄 요약

6가지를 처리한다: ① 레인 하단 이름 표시 ② 시작 시 스크롤 튐 제거(경마식 유지) ③ 추적 속도 느리게 ④ 무작위 순서로 한 명씩 순차 하강 ⑤ 사다리 테스트 봇 추가 ⑥ 내 채팅 가독성 개선.

---

## 1. 레인 밑에 사람 이름 표시

- **무엇**: 사다리 공개/추적 화면에서 각 레인의 **밑(바닥칸) 쪽에 그 레인을 탄 사람 이름**이 보이게 해라. 지금은 토큰 근처(상단)에만 이름이 떠서, 어떤 레인이 누구 것인지 한눈에 안 들어온다.
- **기대**: 각 출발 레인(또는 도착 바닥칸) 하단에 소유자 이름 라벨 고정 표시. 토큰이 내려가는 동안에도 "이 줄 = 누구"가 계속 보인다.
- **결정/위임**: 라벨을 출발 레인 하단에 둘지, 도착 바닥칸 하단에 둘지, 둘 다일지는 가독성 보고 합리적으로. 이름 겹침(레인 많을 때) 회피 처리 포함.

## 2. 시작 시 화면이 위로 튀는 문제 (경마처럼 유지)

- **무엇**: 게임을 시작하면 화면이 위로 스크롤되어 시야가 튄다. **경마처럼 시작 후에도 보던 위치가 유지**되게 해라.
- **작업 방식**: 원인 먼저 조사 — 빌드 섹션 숨김(`display:none`)으로 인한 레이아웃 점프인지, `scrollIntoView`/포커스 이동/`window.scroll*` 호출인지. `js/horse-race.js`가 시작/공개 시 스크롤을 어떻게 다루는지(또는 안 건드리는지) 비교해 동일 동작을 적용.
- **기대**: 사다리 시작→공개 전환에서 사용자의 스크롤 위치가 갑자기 튀지 않는다(부드럽게 캔버스로 유도하는 정도는 허용하되, 경마와 일관).

## 3. 추적(하강) 속도 느리게

- **무엇**: 추적 애니메이션이 너무 빨라 결과를 눈으로 못 따라간다. **전체적으로 느리게** 해라.
- **현재**: `js/ladder.js` `startReveal`의 `DURATION`(현재 약 3200ms)과 자동 종료 타이밍 `LADDER_END_DELAY`(`socket/ladder.js`, 현재 6500ms)가 관여.
- **기대**: 토큰 하강을 충분히 따라갈 수 있는 속도로 조정. 클라 애니 시간 증가에 맞춰 서버 `LADDER_END_DELAY`(공개→자동 종료)도 같이 늘려, 애니가 끝나기 전에 결과 오버레이로 넘어가지 않게 한다(둘이 어긋나면 연출이 잘림).
- **결정/위임**: 구체 수치는 4번(순차 하강)과 합쳐 한 판 총 연출 길이가 과하지 않은 선에서 합리적으로.

## 4. 무작위 순서로 한 명씩 순차 하강

- **무엇**: 지금은 전원 토큰이 동시에 내려간다. **한 명씩, 무작위 순서로 차례차례** 내려가게 해라(한 명 다 내려가면 다음 사람).
- **공정성(중요)**: 하강 순서·연출은 **시각 효과일 뿐 결과와 무관**하다. 패자/매핑은 이미 서버에서 확정된다(`buildLadder`). 순차 연출이 결과를 바꾸지 않음을 보장해라.
- **탭 간 일관**: 순서를 클라마다 다르게 보이게 하지 말 것. **서버 `ladder:reveal`에 하강 순서(또는 시드)를 포함**해 모든 탭이 같은 순서로 보이게 하라(서버에서 셔플 = 서버 RNG, 클라는 받은 순서대로 재생). 클라 `Math.random`으로 순서를 만들지 말 것(공정성 grep 위반 회피).
- **기대**: 무작위 순서로 토큰이 하나씩 내려가고, 각자 도착 시 짧은 강조(꽝 도착자 강조 포함). 마지막에 결과 캡션/오버레이.

## 5. 사다리 테스트 봇 추가

- **무엇**: 사다리 게임을 자동으로 플레이/검증하는 **테스트 봇**을 만들어라. 경마의 테스트 스크립트 패턴을 참고한다.
- **참고**: `AutoTest/` 폴더의 멀티탭 자동화 스크립트(예: `AutoTest/free-multitab-test.js`, `AutoTest/horse-race/test-seed-e2e.js`)와 기존 `tests/test-ladder.js`(Playwright 스모크). 봇/개발도구 코드는 **게임 서버(`server.js`/`routes/`/`socket/`)에 넣지 말고** `AutoTest/`(또는 `tests/`)에 둔다 — 프로젝트 규칙(CLAUDE.md "개발 도구/봇 코드 게임 서버 삽입 금지").
- **기대**: 여러 탭(예: 2~4명)을 자동으로 띄워 방 생성/입장 → 준비 → 빌드 단계에서 레인·막대기 자동 선택 → 시작 → 공개/추적/결과까지 자동 진행. 새 연출(1~4번: 이름 라벨, 스크롤 유지, 느린 속도, 순차 하강)이 깨지지 않는지, 콘솔 에러 없는지 검증. 위치는 `AutoTest/ladder/` 신설 권장.
- **결정/위임**: 풀-자동 봇 1개로 충분한지, 분포/공정성 검증까지 넣을지는 경마 스크립트 구성 보고 합리적으로. 최소한 "여러 명이 한 판 끝까지 플레이"가 자동으로 굴러가야 한다.

## 6. 내 채팅 가독성 개선

- **무엇**: 채팅에서 **내가 쓴 메시지가 너무 안 읽힌다**(배경·글자 대비 낮음). 읽기 쉽게 고쳐라.
- **현재**: 채팅 스타일은 `js/shared/chat-shared.js` + `css/theme.css`의 `.chat-message` 계열(내 메시지/상대 메시지 구분 클래스). 공유 모듈이라 **모든 게임에 영향** — 주사위/룰렛/경마/사다리 채팅 모두에서 확인.
- **기대**: 내 메시지 말풍선의 배경/글자색 대비를 충분히 높여(라이트·다크 양쪽) 가독성 확보. 상대 메시지와 구분은 유지.
- **주의**: `css/theme.css`는 크로스게임 공유 — 색은 CSS 변수로(하드코딩 금지), 다크모드 변수도 함께. 한 게임만 보고 고치지 말고 공통으로 검증.

## 공정성 (전 항목 공통)

- 게임 결과(패자/매핑/꽝)는 **서버에서만** 결정. 1·3·4번 연출은 전부 시각화이며 결과 불변.
- 하강 순서가 무작위여도 **서버 RNG**로 만들어 `ladder:reveal`로 전달. 클라 `Math.random`은 deviceId/tabId 외 0회 유지.
- reveal 전 server-only(숨은 막대기/꽝/매핑) 비노출·재진입 마스킹 유지.

## 작업 방식

- 먼저 `js/ladder.js`(`startReveal`/`drawLadderFrame`/`buildPath`/토큰·이름 렌더), `socket/ladder.js`(`doReveal` reveal 페이로드·`LADDER_END_DELAY`), `js/horse-race.js`(시작 시 스크롤 처리), `js/shared/chat-shared.js`+`css/theme.css`(채팅 스타일)를 조사.
- 4번을 위해 `ladder:reveal` 페이로드에 순서(또는 시드) 필드를 추가하면 클라 `startReveal`가 순차 재생하도록 바꾼다. 서버 트림/마스킹 계약은 유지.
- 모바일/PC 양쪽 레이아웃을 계획 단계부터 포함(이름 라벨 겹침, 캔버스 높이, 채팅 대비).

## 테스트

- `tests/test-ladder.js`(Playwright)를 갱신/확장: 공개 화면에 레인 이름 라벨 존재, 시작 후 스크롤 위치 급변 없음(스크롤 좌표 비교), 순차 하강 동안 캔버스 표시, 결과 도달까지 정상. 콘솔 에러 0.
- 5번 테스트 봇(`AutoTest/ladder/…`)이 한 판 끝까지 자동 플레이.
- 채팅: 내 메시지 가독성 — 최소한 셀렉터/색 대비가 적용됐는지 확인. 기존 대표 게임(경마) 채팅·시작·진행 미파손 확인.

## 완료 기준 (하나라도 미완이면 완료 아님)

- 공개 화면에서 각 레인 밑에 소유자 이름이 보인다.
- 사다리 시작 시 화면이 위로 튀지 않는다(경마와 일관).
- 추적이 충분히 느려 눈으로 따라갈 수 있고, 애니가 끝나기 전에 결과로 넘어가지 않는다.
- 토큰이 무작위 순서로 한 명씩 순차 하강하며, 순서는 모든 탭에서 동일하고 결과는 서버 확정값과 일치한다.
- 사다리 테스트 봇이 `AutoTest/`(또는 `tests/`)에 있고 한 판을 자동으로 끝까지 플레이한다(서버 코드에 봇 삽입 없음).
- 내 채팅 메시지가 라이트·다크 양쪽에서 읽기 쉬워졌고, 다른 게임 채팅이 깨지지 않았다.
- 클라 `Math.random` 게임결과 0회, reveal 전 마스킹 유지.
- update-log.md 기록. 새 리소스가 있으면 함께 기록(가급적 신규 mp3/이미지 없이).
- 기존 대표 게임모드(경마) 미파손 확인.
- 마지막 보고에 항목별 변경 요약, 변경 파일, 테스트 명령/결과, 게임성 자체 평가, 새 리소스 여부, 남은 이슈 포함.

## 막힘 기준

- 이름 라벨 위치, 추적 DURATION/END_DELAY 수치, 순차 하강 간격, 테스트 봇 인원/범위, 채팅 대비 색값 등 세부 불명확 시 기존 구조(경마 포함) 조사 후 근거와 함께 합리적으로 선택.
- 스크롤 튐 원인이 불명확하면 경마와의 차이를 좁히는 방향으로 최소 변경.
- 테스트 환경 문제로 검증 불가 시 구현은 완료하되 어떤 명령/단계에서 막혔는지 구체 보고.
