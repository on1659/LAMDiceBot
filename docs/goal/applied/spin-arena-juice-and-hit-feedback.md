# 회전 칼날(spin-arena) — 타격감/이펙트 폴리시 — Goal

방금 추가한 `spin-arena`(회전 칼날)가 **밋밋해서 재미가 없다.** 칼날이 캐릭터를 갈아도 화면에 아무 일도 안 일어나고(스파크·플래시·흔들림·소리 없음), 캐릭터/칼날이 단색 원·선이라 "타격감"이 0이다. **결정론 리플레이 구조와 공정성을 그대로 유지한 채**, 클라이언트 시각 레이어에 타격 피드백과 이펙트/그림을 입혀 "누가 갈리는지" 보는 재미를 살린다. 새 feature 브랜치 권장.

> 전제/baseline: spin-arena 본체는 완성·검증됨(`docs/meeting/applied/2026-06-10-spin-arena-impl.md`). 서버 결정론 30초 시뮬 → reveal 페이로드(slots/frames(위치·HP 100ms 키프레임 301개)/eliminations/result) → 클라 Canvas 리플레이. **이 구조와 모든 공유 상수는 건드리지 않는다.** 이번 작업은 **클라 렌더 레이어(`js/spin-arena.js` drawSpinFrame + `css/spin-arena.css` + 새 에셋)** 위주.
>
> **확정 방향 (office-hours 2026-06-10, 설계문서: `~/.gstack/projects/on1659-LAMDiceBot/user-feature-ladder-rung-color-speed-design-20260610-125211.md`):**
> - **느낌 = 묵직한 청키 임팩트**(브롤스타즈/모바일 액션 톤). 스파크·플래시·넉백·흔들림·소리로. **데미지 숫자 미표시**("순수 운 관전" 정체성과 충돌). **전역 hitstop 금지**(시간고정 리플레이 desync). 슬로우모션은 허용하되 **추가 지연 ≤ RESULT_HOLD_MS(2.2초)**(서버 endTimeout 32.2s 침범 금지).
> - **범위 = 풀 비주얼 오버홀(B) 전부를 한 작업으로 완성**(증분 출시 아님 — 사용자 지시). ① 이펙트 코드(bridge-cross 재활용) + ② 스프라이트 아트(SpriteMake 생성) + ③ 카메라/연출(트레일·최후 결판 줌/슬로우·승자)를 **한 PR에 모두** 포함. ①②③ 번호는 구현 순서 가이드일 뿐, 부분 출시·후속 미룸 없이 한 번에 끝낸다.
> - **타격 이펙트 게이트 = (칼날끝↔몸 겹침, frames 원본좌표, 임계값 (14+7)²=441) AND (보간 HP 실제 감소 중)** — 최후 1인 무적 구간에서 스파크 남발 방지. 넉백은 렌더 출력에만(원본좌표 불변).

## 한 줄 요약

① **타격 피드백 스택** — 칼날이 몸에 닿을 때(HP 감소 순간) 스파크·플래시·HP바 반응·소리·미세 흔들림으로 "맞는 느낌"을 준다 ② **탈락/링/결판 연출** — 탈락 시 큰 파편+화면흔들림+충격파, 링 밖 위험(붉은 비네트), 최후 결판 줌/집중 ③ **그림 업그레이드** — 단색 원/선 → 스프라이트 또는 리치 프로시저럴(캐릭터·칼날·아레나) ④ **리소스** — bridge-cross 기존 이펙트 코드/스프라이트 재활용 우선, 부족하면 `D:\Work\vibe\SpriteMake`로 spin-arena 전용 생성.

## 핵심 규칙

### 1. 타격 피드백 스택 (칼날-몸 접촉 = HP 감소 순간) — 가장 중요
지금 가장 큰 결함: **리플레이 중 HP가 줄어도 화면에 아무 반응이 없다.** 매 프레임 HP 감소(또는 클라가 위치+칼날각으로 직접 계산한 칼날끝-몸 겹침)를 감지해 다음을 트리거:
- **임팩트 스파크** — 접촉 지점(칼날끝↔몸 사이)에 가해자 칼날색 파티클 짧게 튐.
- **피격 플래시** — 맞은 캐릭터 몸이 흰색→스킨색으로 ~120ms 점멸.
- **넉백 저크(시각 전용)** — 맞은 캐릭터를 충격 반대쪽으로 살짝 밀었다 복귀(렌더 오프셋만, **서버 위치는 불변** — 리플레이 좌표 자체는 안 건드림).
- **HP바 펀치** — 바가 짧게 흔들리고, 줄어든 만큼 흰색 "칩 데미지" 잔상 바가 뒤따라 빠짐.
- **타격음** — `spin-arena_hit`, 작은 볼륨 + 강도별 약간의 피치 변주. **초당 호출 상한(throttle)** 필수(50회/초 난사 금지).
- **미세 화면 흔들림** — 강한/연속 타격에만, 아주 작은 진폭.

### 2. 탈락 연출 (지금 death-burst보다 크고 명확하게)
- 화면 흔들림(bridge-cross `Camera.shake` amp~10–14 패턴), 스킨색 **파편 폭발**(현재 8조각보다 풍성하게), 탈락 지점에서 퍼지는 **충격파 링**, 짧은 채도 저하/플래시, `spin-arena_eliminate` 사운드, "OUT"/순위 플로팅 텍스트.

### 3. 긴장 연출 (앰비언트)
- **안전 링**: 점선 회전 + 글로우 펄스, 수축할수록 펄스 빨라짐. **링 밖**: 붉은 비네트 + 위험 파티클, 링 밖 캐릭터 붉은 틴트.
- **칼날 트레일**: 회전 칼날이 페이드되는 잔상 호(모션블러) — 위협감.
- **최후 결판(2~3인)**: 가장자리 비네트 어둡게 + 중심으로 미세 줌/집중, 긴장 사운드(선택).
- **승자**: t=30s 생존자에 글로우/왕관/승리 버스트.

### 4. 그림(스프라이트) 업그레이드
- **캐릭터**: 단색 원 → 스프라이트 또는 리치 프로시저럴(그라데이션 바디 + 림라이트 + 그림자 + 간단한 표정/눈). **6스킨 색 정체성 유지**(스킨은 색 기반 — 공정/외형 전용), 봇은 회색.
- **칼날**: 선/삼각형 → 금속 광택 칼날(스프라이트 또는 프로시저럴) + 회전 글린트.
- **아레나**: 바닥 텍스처/서브틀 그리드/비네트, 위험 구역 시각 구분.

### 5. 리소스 소스 (결정 위임 — 아래 우선순위)
- **1순위 — bridge-cross 재활용**: `assets/bridge-cross/sprites`, `js/bridge-cross.js`의 `Camera.shake`(1364행대) + 스프라이트시트 애니 플레이어 + `safe_sparkle`/`crack`/`safe-flash` 시트가 검증돼 있음. **이펙트 코드 패턴을 먼저 lift**하고, 스파크/파편 시트가 시각적으로 맞으면 그대로 차용.
- **2순위 — SpriteMake 생성**: `D:\Work\vibe\SpriteMake`(README/USAGE 참조, `output/`에 bridge-cross·horse 등 생성 이력). 임팩트 스파크 시트/파편 조각/캐릭터 바디/칼날 글린트를 spin-arena 전용으로 생성 → `assets/spin-arena/`로 복사.
- **권장**: 흔들림/플래시/스파크 같은 **이펙트는 코드+기존 시트로** 빠르게 깔고, **캐릭터/칼날/아레나 스프라이트 아트(SpriteMake 생성)까지 이번 작업에 포함**한다. 프로시저럴로 충분한 부분은 프로시저럴, 부족한 부분은 SpriteMake — **둘 다 이번에 완성**(아트를 후속으로 미루지 않는다).

### 6. 설계 메모 — "타격감"을 만드는 것 (game juice 원칙, 이 게임 맥락)
- 이 게임은 **시간 고정 리플레이**다 → 모든 이펙트는 **시각 전용**, 리플레이 데이터(위치/HP/탈락)에서 파생, 타임라인을 절대 바꾸지 않는다.
- **진짜 hitstop(게임 시간 정지)은 피한다** — 리플레이 시계를 멈추면 탭별 desync 위험. 대신 **피격자 스케일-펀치 + 플래시**로 "멈칫" 느낌만. (전역 hitstop이 꼭 필요하면 결정론적으로 동기화 가능한지 구현자가 판단·보고.)
- 좋은 타격감 = **다중 채널 동시 자극**(시각 스파크 + 색 플래시 + 위치 저크 + 사운드 + 약한 흔들림)이 **짧고 즉각적**으로. 과하면 피로 → 강도별 차등(약타=스파크만, 강타/탈락=풀스택).
- 더 깊은 탐색을 원하면 구현 전에 `/office-hours` 또는 `/design-consultation`/`/design-review`로 발전시켜도 좋다. 단 이 goal은 위 스택만으로도 바로 착수 가능하게 구체적으로 적었다.

## 공정성 (절대 불변)
- **서버 결정론 시뮬·reveal 페이로드·공유 상수 일체 변경 금지.** frames/eliminations/result 동일. 누가 당첨인지는 서버에서만 결정.
- 모든 이펙트는 **클라 시각 전용**, 리플레이 데이터에서 파생 → 전 탭 일관. 넉백/스케일-펀치는 **렌더 오프셋만**, 리플레이 좌표·충돌·결과 불변.
- **클라 Math.random**: 결과 영향 0회 유지. **흔들림/파티클 지터 같은 외관 효과의 cosmetic random은 허용**(`.claude/rules/new-game.md` §7 명시). 단 공정성 grep(`grep -c "Math.random" js/spin-arena.js`)의 의미는 깨지지 않게 — 결과 결정 경로엔 절대 안 들어간다. (현재 deviceId/tabId 2회 + 결정론 파티클. cosmetic shake로 늘어나면 그 사실을 보고에 명시.)
- reveal 전 server-only(timeline/result/seed) 비노출·재진입 마스킹 그대로.

## 막힘 기준
- 이펙트 강도/파티클 수/스프라이트 스타일 등 수치·형태가 애매하면 bridge-cross 선례 조사 후 가장 단순·공정한 쪽으로 정하고 보고에 명시.
- 타격 감지를 클라 자체 계산(위치+칼날각 겹침) vs HP-감소 관찰 중 어느 쪽으로 할지는 구현자가 단순·정확한 쪽 선택(서버 페이로드 변경이 필요하면 그땐 소켓 영향 = 트리아지 상향 + 보고).
- 스프라이트 아트는 **이번 범위에 포함**이다(임의로 후속 미루지 말 것). SpriteMake가 도구/시간으로 **완전히 막힐 때만** 그 부분을 프로시저럴로 메우고, 막힌 지점·사유를 구체 보고.
- 테스트 불가(서버 미기동 등) 시 구현은 완료하되 어디서 막혔는지 + 수동 QA 체크리스트.
