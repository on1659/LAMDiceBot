# 회전 칼날(spin-arena) — 중앙 군집 + 링 이탈 금지 검토 + 칼날 히트박스 — Goal

spin-arena 전투 체감 후속 3건: ① 캐릭터들이 서로 모이지 않는 문제 — 센터점 기준으로 모이게 ② 링 밖 HP 감소 대신 아예 못 나가게 하는 안 검토 ③ 타격 판정을 "막대 끝 점"에서 "칼날(검) 구간"으로 확장. 셋 다 서버 시뮬 변경이므로 **밸런스 재튜닝(200시드 배치)이 필수 게이트**. 브랜치는 `feature/spin-arena-rework` 이어서 작업.

> 전제/baseline: 2026-06-11 게임성 개편 완료 상태(`docs/goal/applied/spin-arena-gameplay-rework-and-parity.md` — 봇 제거/가변 n, 킬당 칼날 +1 상한 5, 서버 실제 넉백 kvx/kvy, 검 그래픽, 카운트다운/튜토리얼/BGM/다시보기). 현재 튜닝값: DRIFT_SPEED 50, INWARD_ACCEL 150, HIT_DPS 110, RING_DPS 12, RING_R_END 60, KNOCK 70/110/3.0. 충돌 판정 = 칼끝 점 vs 몸 원 (`(CHAR_RADIUS+BLADE_TIP_R)² = 441`).

## 한 줄 요약

① 중앙 인력(센터 시킹)으로 캐릭터들이 자연히 모여 싸우게 ② 링 이탈을 막는 하드 월 채택 여부를 배치 시뮬로 검토·결정 ③ 히트박스를 칼끝 점 → 검 날 구간(선분)으로 — 보이는 칼이 곧 타격 범위.

## 1. 중앙 군집 (센터 시킹)

- 현상: 현재 이동은 랜덤 드리프트 + 링 밖일 때만 INWARD_ACCEL → 서로 흩어져 30초 내내 안 만나는 판이 생긴다.
- 요구: **센터점(CX, CY) 기준으로 모이게** — 링 안에서도 항상 중심 방향으로 약한 인력(상수 예: `CENTER_PULL`)이 걸리게 한다. 드리프트의 무작위 방향성은 유지하되(완전 일직선 돌진 금지 — 밋밋해짐), 시간이 갈수록 중심 근처 밀도가 올라가 칼날 교전이 자연 발생해야 한다.
- 전부 결정론 시뮬 내부(시드 PRNG). 세기는 A절 배치 튜닝으로 확정.

## 2. 링 이탈 금지 검토 (결정 위임 — 검토 후 채택 여부 보고)

- 제안: 링 밖에서 HP가 닳는 현재 방식 대신, **링 경계를 하드 월로 만들어 아예 못 나가게** (넉백·드리프트로 밀려도 경계에서 클램프/반사).
- 채택 시 효과: 링 수축이 "압축 장치"가 되어 1번(군집)과 시너지 — 후반 강제 근접전. 킬이 전부 칼날 귀속이 되어 신메커니즘(킬 성장)이 살아난다. 넉백은 "링 밖으로 밀어내기"에서 "벽에 몰아붙이기"로 의미 변경.
- 리스크: 링 데미지가 사라지면 30초 내 결판률이 무너질 수 있다(현재 h2 98%는 링 데미지 의존). **판정 기준 = 200시드 배치**: 하드 월 적용 후 h2/h3 ≥ 95%, h6 ≥ 88% "정확히 1명 생존"을 유지 못 하면 보완 레버(벽 접촉 중에만 소량 데미지, HIT_DPS 상향, 수축 타이밍 단축) 순으로 조정. 그래도 미달이면 하드 월 폐기하고 현행 유지 — **결정과 근거(분포 수치)를 보고에 명시**.
- 채택 시 RING_DPS 관련 코드는 깔끔히 정리(레버로 남기는 경우는 주석으로 의도 명시), 클라의 링 밖 빨간 데미지 연출·튜토리얼 문구도 새 규칙에 맞게 갱신.

## 3. 칼날 히트박스 — 점 → 검 날 구간

- 현상: 검 그래픽(도신+가드+힐트)은 들어갔지만 **판정은 여전히 칼끝 점 하나** — 보이는 칼 중간에 맞아도 노히트.
- 요구: 타격 범위 = **칼날(도신) 구간**. 서버 시뮬에서 칼끝 점 충돌을 "날 선분(허브에서 `BLADE_RADIUS - SWORD_LEN` ~ `BLADE_RADIUS` 구간) vs 몸 원" 충돌로 교체 — 선분-원 거리 판정, 전부 결정론.
- **보이는 것 = 맞는 것**: 클라 검 그래픽의 도신 길이를 서버 `SWORD_LEN`과 같은 공유 상수로 맞춘다(서버·클라 상단 const 동기). 타격 스파크 위치도 실제 접촉점 기준으로.
- 클라 타격감지(이펙트용 tips 근접 검사)도 같은 선분 판정으로 동기 — 안 맞으면 스파크와 실데미지가 어긋난다.
- 판정이 후해지므로(접촉 시간 증가) HIT_DPS·넉백 재튜닝 필요 — A절 배치 게이트.

## A. 밸런스 재튜닝 (필수 게이트)

- 1~3 적용 후 **200시드 배치**(`AutoTest/spin-arena-determinism-test.js` 내 배치 루프)로 h2~h6 "정확히 1명 생존" 분포 재검증: **h2/h3 ≥ 95%, h6 ≥ 88%**.
- **칼날킬 비율도 함께 보고** (직전 작업 교훈: 생존률만 보면 환경 데미지가 킬을 독식해도 통과한다). 하드 월 채택 시 칼날킬 ~100%가 정상.
- 조정 레버: CENTER_PULL / SWORD_LEN / HIT_DPS / KNOCK_* / 링 수축 타이밍 / DRIFT_SPEED. 기준 충족하는 가장 단순한 조합 선택, 최종 수치 표를 보고에 첨부.
- 결판 규칙(사람 최후 1인 무적·전체 1명 정지)·survivalKey 랭킹·selected(당첨) 시맨틱·DB 기록 불변.

## 공정성 (절대 불변)

- 결과는 서버 결정론 시뮬에서만(시드 PRNG, 동일 시드 = 동일 frames/eliminations/result). 군집 인력·하드 월·선분 판정 전부 시뮬 내부.
- 클라 Math.random 실호출 2회(deviceId/tabId) 유지. 연출은 전부 t·페이로드·hash01 파생(2탭 동일).
- reveal 전 server-only(timeline/result/seed) 비노출·재진입 마스킹 유지. 페이로드 변경이 필요하면 additive만.

## 기존 통합 유지 (스킵 금지)

- 킬당 칼날 +1(상한 5)·killerId·데미지 숫자·HP 패널·채팅 오버레이·카운트다운·튜토리얼·BGM·다시보기 전부 계속 동작.
- 소켓 이벤트 이름 유지(requestSkins 포함 9개). 주문/준비/통계/랭킹/히스토리/결과 오버레이 유지.
- 튜토리얼·update-log 문구가 규칙 변경(특히 2번 채택 시)과 모순되지 않게 갱신.

## 작업 방식

- 먼저 읽을 것: `socket/spin-arena.js` simulate(이동/충돌/넉백 루프), `js/spin-arena.js`(drawBladeSet 검 그래픽 치수, 타격감지 tips, 링 연출), `docs/goal/applied/spin-arena-gameplay-rework-and-parity.md`(직전 결정·계약).
- 서버·클라 공유 상수(SWORD_LEN 등)는 양쪽 파일 상단 const 블록 동시 수정 — 값 다르면 데싱크.
- 캐시버스트: `js/spin-arena.js` 로드 쿼리(`?v=2`)를 올린다(상수 변경 배포 시 구클라 desync 방지).
- **소켓 변경 → dev 서버(5173) 재시작 필수** (자동 리로드 없음).
- 모바일·PC 양쪽 확인. 캔버스 논리 480×480 유지.

## 테스트

- `node -c socket/spin-arena.js` + `node --check js/spin-arena.js` + 공정성 grep(Math.random 실호출 2회).
- 결정론 회귀: `AutoTest/spin-arena-determinism-test.js` — 동일 시드 동일 결과, frames 301·폭 n×3 유지. 선분 판정/하드 월 반영해 필요한 단언 갱신.
- 200시드 배치 분포 + 칼날킬 비율 표 (A절 기준 충족 수치 첨부).
- 렌더 하네스(`AutoTest/spin-arena-render-harness.html`) `__PAYLOAD` 재생성 → 실브라우저 스모크(에러 0, 검 길이 = 판정 길이 시각 확인, 같은 t 2회 픽셀 동일).
- 로컬 5173 + 2탭(`AutoTest/spin-arena-2tab-test.js`): 동일 reveal/당첨자, 군집 체감(중반 이후 중앙 교전), 하드 월 채택 시 링 밖 이탈 0 확인. 경마/사다리/주사위/룰렛 미파손.

## 완료 기준 (하나라도 미완이면 완료 아님)

- 1·3 구현 + 2 검토 결정(채택/폐기 + 분포 근거) 완료.
- 200시드 분포 기준 충족(h2/h3 ≥95%, h6 ≥88%) + 칼날킬 비율 보고.
- 보이는 검 = 타격 범위 (공유 상수로 검증 가능).
- 결정론 회귀·2탭 동일 통과. 클라 Math.random 2회 유지.
- 규칙 변경 시 튜토리얼 문구 동기 갱신.
- update-log.md 기록(평이한 한국어). 새 리소스 없음 예상 — 생기면 명시.
- 마지막 보고: 변경 요약·파일·테스트 명령/결과·튜닝 최종 수치 표·2번 결정과 근거·자체 평가·남은 이슈.

## 막힘 기준

- CENTER_PULL 방식(상시 인력 vs 시간 비례 증가 vs 페이즈 연동), SWORD_LEN 수치, 하드 월의 클램프/반사 선택은 구현자 판단 — 배치 분포가 권위, 결정과 사유만 보고.
- 하드 월 채택 시 넉백 상수가 무의미해지면(벽이 다 막음) KNOCK_* 재조정 또는 벽 슬라이드 등 가장 단순한 쪽 선택 후 보고.
- 테스트 불가 시 구현은 완료하되 막힌 지점 + 수동 QA 체크리스트.
