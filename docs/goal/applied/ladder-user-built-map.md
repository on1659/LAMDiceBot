# 사다리타기 유저 빌드 맵 — Goal

사다리타기(ladder) 게임을 "유저가 맵을 직접 만드는" 방식으로 확장해라. feature/goal-game-mode 브랜치에서 작업한다.

## 핵심 규칙

- 기존: 서버가 막대기(가로줄)를 전부 난수로 생성 → 레인 선택 → 공개. 이 "서버 전체 난수 생성"을 "가려진 기본 틀 + 유저 빌드"로 대체한다.
- 가려진 기본 틀: 서버가 소수의 "숨은 막대기"를 미리 깔아둔다(reveal 전까지 클라에 전송 금지).
- 유저 빌드: 경마의 탈것 선택과 동일 — 준비(ready)를 눌러야 막대기를 놓을 수 있고, 놓는 건 선택이다(준비만 하고 안 놓아도 됨). 1인당 최대 1개, 행 + 인접 두 열 사이를 클릭해 위치를 직접 고른다.
- 시작 조건: 호스트는 준비 인원이 충분하면 막대기를 안 놓은 참가자가 있어도 시작할 수 있다(배치 강제 안 함).
- 소유권/취소: 각 막대기는 배치한 사람 소유다. 그 사람이 준비를 취소하거나 방을 나가면, 그 사람이 넣은 막대기는 사라진다.
- 레인 선택 유지: 막대기 배치가 끝난 뒤 기존처럼 각자 레인을 1개 고른다(중복 불가). 막대기 배치와 레인 선택 둘 다 선택 요소로 유지한다.
- 가시성/공정성: 유저가 놓은 막대기는 모두에게 실시간으로 보인다. 단 바닥 꽝 위치와 서버 숨은 막대기는 reveal 전까지 숨긴다. 재진입 응답에서 server-only 정보(숨은 막대기/꽝/매핑) 마스킹을 유지한다.
- 결과 결정: 최종 사다리(기본 틀 숨은 막대기 + 유저 막대기)를 따라 토큰이 내려가 꽝 도착자가 패자다. 결과는 서버에서만 결정하고 클라는 시각화만 한다. 클라 Math.random은 deviceId/tabId 외 0회.
- 같은 행에서 인접한 막대기는 금지(표준 사다리 제약)하여 추적이 모호하지 않게 서버가 검증한다. 클라 추적 알고리즘과 서버 매핑이 일치해야 한다.

## 작업 방식

- 먼저 현재 socket/ladder.js, js/ladder.js, css/ladder.css, ladder-multiplayer.html, utils/room-helpers.js의 ladder gameState, socket/rooms.js cleanup/마스킹, socket/shared.js 준비(toggleReady/setUserReady) 처리를 조사하고 .claude/rules/new-game.md를 참조해라.
- 막대기 배치/제거 소켓 이벤트(ladder:addRung {row,col} / ladder:removeRung / 브로드캐스트 ladder:rungsUpdated)를 추가하고, 서버가 위치 유효성(범위·인접 금지·1인 1개·준비 여부·소유권)을 검증하게 해라.
- 준비 토글 취소와 leaveRoom/disconnect 시 해당 유저 막대기 정리를 socket/shared.js(준비 분기, 경마 패턴 참고)·socket/rooms.js·ladder 핸들러에 연결해라.

## 게임성 기준

- 한 판 흐름이 짧고 명확. 운(서버 숨은 막대기+꽝 숨김)과 선택(막대기 위치+레인) 모두 존재. 결과 즉시 이해(꽝=패자). 반복 플레이 이유(매 판 새 기본 틀). 완료 보고에 재미/선택/긴장/반복 자체 평가 포함.

## 리소스 기준

- 기존 리소스/사운드 우선 재사용. 막대기 효과음은 기존 ladder_*/common_* 재사용. 신규 mp3/이미지는 꼭 필요할 때만 최소로. 기존 사운드 파일명 변경 금지. 새 리소스는 update-log에 기록.

## 기존 통합 유지/갱신(스킵 금지)

- 통계(db/stats, /api/statistics), 랭킹(db/ranking getFullRanking + RankingModule ladder 탭) 계속 동작.
- 튜토리얼: LADDER_TUTORIAL_STEPS에 "막대기 놓기" 단계 추가 + 최초 1회 자동 노출 + 도움말 버튼 동작.
- 사운드: 막대기 배치/공개/결과 등 핵심 순간 재생 호출 연결, 로드 실패 없음.

## 테스트

- tests/test-ladder.js(Playwright)를 새 규칙에 맞게 갱신: 앱 실행 → 방 생성/입장(2탭) → 준비 → 각자 막대기 1개 배치(브로드캐스트 동기화) → 준비 취소 시 본인 막대기 제거 확인 → (한 명은 미배치) 레인 선택 → 공개/추적 → 결과/종료 도달 → 콘솔 에러 없음.
- 통계 집계, 랭킹 ladder 섹션, 튜토리얼 자동 노출도 검증. 기존 대표 게임(경마) 선택/시작/진행 가능 확인.

## 완료 기준(하나라도 미완이면 완료 아님)

- 준비한 유저가 막대기를 선택적으로 배치(미배치 허용)해 맵을 만들 수 있고, 준비 취소/퇴장 시 본인 막대기가 사라진다.
- 막대기 배치 → 레인 선택 → 공개/결과/종료 흐름이 실제로 동작한다.
- 꽝/서버 숨은 막대기가 reveal 전 클라에 노출되지 않는다(재진입 마스킹 포함). 클라 Math.random 게임결과 0회.
- 통계/랭킹/튜토리얼/사운드가 모두 동작한다.
- 기존 대표 게임모드 1개 이상이 깨지지 않았음을 확인했다.
- update-log.md에 기존 형식대로 기록.
- 마지막 보고에 룰 요약, 변경 파일, 테스트 명령, 테스트 결과, 게임성 자체 평가, 새 리소스 여부, 남은 이슈 포함.

## 막힘 기준

- 열 개수 산정, 행 수, 숨은 기본 막대기 개수, 빌드 종료 시점 등 세부 불명확 시 기존 구조 조사 후 근거와 함께 합리적으로 선택.
- 테스트 환경 문제로 검증 불가 시 구현은 완료하되 어떤 명령/단계에서 막혔는지 구체 보고.
