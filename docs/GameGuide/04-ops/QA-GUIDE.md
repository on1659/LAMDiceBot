# QA 지침서

코드 변경 뒤 최소한으로 확인해야 하는 기준을 정리한 문서입니다.

이 문서는 "현재 저장소에 실제로 있는 도구와 경로"만 기준으로 작성합니다.

## 기본 원칙

- 문법 확인 -> 서버 부팅 확인 -> 브라우저 콘솔 확인 순서로 진행합니다.
- 공통 모듈을 건드렸다면 주사위, 룰렛, 경마를 모두 확인합니다.
- 콘솔 에러가 하나라도 보이면 원인 분석 없이 통과 처리하지 않습니다.

## QA 단계

### 1. 정적 확인

- `node -c server.js`
- 변경 파일 기준으로 경로, 이벤트명, 함수명 오타 확인
- 삭제되었거나 이동한 문서를 참조하지 않는지 확인

### 2. 서버 확인

- `node server.js`
- 부팅 직후 치명적 예외가 없는지 확인
- 주요 라우트가 200 또는 의도한 리다이렉트를 반환하는지 확인

주요 공개 경로:

- `/game`
- `/horse-race`
- 필요 시 `/roulette`

### 3. 브라우저 확인

- 페이지 최초 로드 시 콘솔 에러가 없는지 확인
- 방 생성/입장 시 콘솔 에러가 없는지 확인
- 변경한 버튼/폼/소켓 흐름을 직접 한 번 이상 실행

특히 자주 보는 회귀 포인트:

- `document.hasAttribute` 같은 잘못된 DOM API 사용
- 초기화 중 예외로 인해 화면 일부가 비어 보이는 현상
- 공통 모듈 수정 후 특정 게임만 정상 동작하는 상태

### 4. 공통 모듈 영향 확인

아래 파일을 수정했다면 최소 3개 게임을 함께 확인합니다.

| 모듈 | 위치 | 영향 범위 |
|------|------|-----------|
| `ready-shared.js` | `js/shared/` | 주사위, 룰렛, 경마 |
| `chat-shared.js` | `js/shared/` | 주사위, 룰렛, 경마 |
| `order-shared.js` | `js/shared/` | 주사위, 룰렛, 경마 |
| `ranking-shared.js` | `js/shared/` | 주사위, 룰렛, 경마 |
| `control-bar-shared.js` | `js/shared/` | 주사위, 룰렛, 경마 |
| `server-select-shared.js` | `js/shared/` | 주사위, 룰렛, 경마 |
| `server.js` | 루트 | 전체 서비스 |

## 자동화 테스트

### 현재 사용 가능한 스크립트

- 경마 슬로모션 검증: `node AutoTest/horse-race/test-loser-slowmo.js`

### 현재 저장소에 없는 예전 도구

아래 항목은 예전 문서에는 남아 있을 수 있지만, 지금 저장소에는 없습니다.

- `dice-test-bot.js`
- 예전 roulette test bot
- 예전 console error check 스크립트

따라서 자동화가 부족한 영역은 수동 브라우저 확인을 기본으로 잡아야 합니다.
