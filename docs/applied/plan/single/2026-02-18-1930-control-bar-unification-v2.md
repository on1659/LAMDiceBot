**일시**: 2026-02-18
**주제**: 상단 컨트롤바 통일 v2 (레이아웃 + 볼륨 슬라이더 복원)
**참석자 (관점)**:
- 기획팀: 기획 팀장, 기획 사원
- 개발팀: 개발 팀장, 개발 사원
- 디자인팀: UI/UX 디자이너, 사운드 디자이너
- 품질팀: QA 엔지니어
**회의 방식**: 1인 순차 분석 (경량 버전)
**게임 대상**: 주사위 / 룰렛 / 경마 / 인형뽑기 (공통)

## 1. 현황 요약

### 문제점

1. **HTML 구조가 게임마다 다름**: 같은 `.room-control-bar`를 사용하지만 내부 요소가 제각각
   - 주사위: `roomStatusIcons` (IP 차단 아이콘), `currentUser` span
   - 룰렛: `turboBadge` (터보 뱃지), `usernameDisplay` span
   - 경마/인형뽑기: `usernameDisplay` span (주사위와 다른 id)

2. **사운드 컨트롤 다운그레이드**: 원래 볼륨 슬라이더(0~100)였으나 체크박스(on/off)로 변경됨
   - 룰렛/인형뽑기: JS에 볼륨 슬라이더 코드가 남아있지만 HTML은 체크박스
   - 주사위: 완전히 체크박스 전용 코드
   - 경마: 사운드 컨트롤 JS 자체가 없음 (volume 관련 코드 0건)

3. **"사운드" 텍스트 줄바꿈**: 세로로 좁은 공간에서 "사운" + "드"로 깨짐

4. **방제목 가운데 정렬 밀림**: 같은 줄에 HOST 뱃지, 터보 뱃지, 유저명이 있어 방제목이 왼쪽으로 밀림

## 2. 관점별 분석 결과

### 기획팀

#### 2-1. 기획 팀장 (A1)
- **볼륨 슬라이더 복원은 필수**: on/off만으로는 사용자 경험 후퇴. "소리를 작게 하고 싶은" 니즈가 명확함
- **컨트롤바를 2줄 구조로 전환**: 1줄 방제목 + 2줄 메타정보(뱃지, 볼륨, 나가기)로 분리하면 깔끔
- **우선순위**: 볼륨 복원 > 레이아웃 정리 > 텍스트 제거

#### 2-2. 기획 사원 (A2)
- 유저 입장에서 "볼륨 슬라이더가 사라졌다"는 기능 퇴보로 느껴짐
- 방 제목이 크고 가운데에 있으면 "내가 어디 있는지" 한눈에 파악 가능
- 뱃지류(HOST, 터보)는 방제목 아래에 작게 표시하면 정보 과부하 감소

### 개발팀

#### 2-3. 개발 팀장 (B1)
- **핵심 문제**: 컨트롤바 HTML이 4개 파일에 각각 하드코딩. 하나 수정하면 4곳 다 수정해야 함
- **현실적 해법**: 공유 JS 파일(`control-bar-shared.js`)로 컨트롤바를 동적 생성하면 한 곳만 수정하면 됨
- 단, 지금 당장은 오버엔지니어링. 4개 파일 HTML을 동일하게 맞추고, CSS로 레이아웃을 통일하는 게 우선

#### 2-4. 개발 사원 (B2)
- **현재 상태 정리**:
  - 룰렛/인형뽑기: `volumeBtn` + `volumeSlider` JS 코드 있음 → HTML에 슬라이더 복원하면 바로 동작
  - 주사위: 볼륨 슬라이더 JS 없음 → 새로 추가 필요
  - 경마: 볼륨 관련 코드 자체 없음 → 새로 추가 필요
- **구현 방향**:
  1. HTML 컨트롤바를 2줄 구조로 통일 (4개 파일)
  2. 볼륨 슬라이더를 공통 패턴으로 추가 (4개 파일)
  3. CSS는 `theme.css`의 `.room-control-bar`만 수정

### 디자인팀

#### 2-5. UI/UX 디자이너 (C)
- **2줄 레이아웃 제안**:
  ```
  ┌─────────────────────────────────────────────┐
  │          🎲 방 제목 ✏️                       │  ← 1줄: 방제목만 (가운데)
  │  HOST  터보  닉네임   🔊━━━●━━  🚪나가기    │  ← 2줄: 메타정보 + 액션
  └─────────────────────────────────────────────┘
  ```
- 볼륨 슬라이더: 아이콘(🔊) + 짧은 슬라이더(60px). "사운드" 텍스트 제거
- 나가기 버튼: 작게 유지 (현재 크기 OK)
- 모바일에서는 2줄 그대로 유지하되 슬라이더 너비만 줄임

#### 2-6. 사운드 디자이너 (D)
- 볼륨 조절은 필수 기능. 게임마다 효과음 크기가 다르므로 유저가 조절할 수 있어야 함
- 볼륨 값은 `localStorage`에 게임별로 저장 (현재 룰렛/인형뽑기는 이미 구현됨)
- 경마와 주사위에도 동일한 볼륨 시스템 추가 필요

### 품질팀

#### 2-7. QA 엔지니어 (E)
- **테스트 방법**: 4개 게임 동시에 열어서 컨트롤바 스크린샷 비교
- 볼륨 슬라이더: 0→50→100 드래그 테스트, 뮤트 토글 테스트, 새로고침 후 값 유지 확인
- 모바일: iOS Safari, Android Chrome에서 슬라이더 터치 동작 확인
