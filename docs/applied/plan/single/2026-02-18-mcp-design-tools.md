# LAMDiceBot 팀 회의록 (Single)

**일시**: 2026-02-18
**주제**: 새로운 디자인 요소를 만들 수 있는 MCP 도구 필요성 전수조사
**참석자 (관점)**:
- 기획팀: 기획 팀장, 기획 사원
- 개발팀: 개발 팀장, 개발 사원
- 디자인팀: UI/UX 디자이너, 사운드 디자이너
- 품질팀: QA 엔지니어
**회의 방식**: 1인 순차 분석 (경량 버전)
**게임 대상**: 주사위 / 룰렛 / 경마 / 팀 배정 / 공통

---

## 1. 현황 요약

### 현재 디자인 리소스 현황

**보유 리소스**:
- **사운드**: 게임별 BGM + 효과음 (MP3/WAV) — 총 20개 파일
  - common: button_click, countdown, notification, nosound
  - dice: bgm, result, roll, win
  - roulette: bgm, spin, stop, winner
  - horse-race: bgm, countdown, cheer_burst, crowd, finish, gunshot, result, slowmo
  - team: bgm
- **배경 이미지**: PNG 기반 7개 테마 (beach, expressway, forest, ocean, road, sky, space)
- **캐릭터 시스템**: 이모지 기반 11개 (🚗🚀🐦🚤🚴🐇🐢🦅🛴🚁🐎)
- **색상 시스템**: theme.css — Material Design 팔레트 (140개 CSS 변수)
- **테마 설정**: vehicle-themes.json (11개 테마 매핑)

**부족한 리소스**:
1. **커스텀 그래픽** — 이모지 의존, 브랜드 정체성 부족
2. **SVG 아이콘** — UI 컴포넌트용 아이콘 없음
3. **스프라이트 애니메이션** — 정적 이미지만 존재
4. **브랜드 로고/아이콘** — 프로젝트 아이덴티티 부재
5. **게임별 일러스트** — 게임 분위기 강화 요소 부족
6. **로딩/전환 애니메이션** — 시각적 피드백 제한적

**최근 작업 내역**:
- 컨트롤바 통일 v3 완료 (2026-02-17)
- UI 색상 시스템 정리 완료 (theme.css 140 variables)
- 이모티콘 설정 시스템 개선 (JSON 기반)

## 2. 관점별 분석 결과

### 기획팀

#### 2-1. 기획 팀장 (A1) — 전략적 방향

**제안 1: 브랜드 아이덴티티 MCP 도구 (우선순위: 높음)**
- **목적**: LAMDiceBot 고유 브랜드 정체성 확립
- **필요 기능**:
  - 로고 생성 (SVG 기반, 다크/라이트 모드 대응)
  - 파비콘 생성 (16x16 ~ 512x512 멀티 해상도)
  - 오픈그래프 이미지 생성 (소셜 미디어 공유용)
- **비즈니스 가치**: 브랜드 인지도 ↑, 신뢰도 ↑
- **우선순위 근거**: 현재 프로젝트가 단순 "게임 사이트"로 인식됨. 독자적 아이덴티티 부재.

**제안 2: 게임 테마 일러스트 생성 MCP (우선순위: 중간)**
- **목적**: 각 게임 모드별 전용 일러스트로 몰입감 강화
- **필요 기능**:
  - 프롬프트 기반 이미지 생성 (DALL-E, Midjourney API 연동)
  - 스타일 일관성 유지 (프로젝트 테마 색상 팔레트 준수)
  - PNG/WebP 자동 변환 + 압축
- **비즈니스 가치**: 게임별 차별화 ↑, 재방문율 ↑
- **우선순위 근거**: 현재 배경 이미지는 있으나 "게임 전용" 일러스트 부족.

**제안 3: 동적 OG 이미지 생성 API (우선순위: 낮음)**
- **목적**: 방 공유 시 게임 상태를 담은 OG 이미지 자동 생성
- **필요 기능**:
  - 실시간 게임 상태 (참가자 수, 게임 타입) → OG 이미지
  - Vercel OG Image API 스타일 (satori 라이브러리)
- **비즈니스 가치**: 공유 유입 ↑, 바이럴 효과
- **우선순위 근거**: 현재 멀티플레이어 중심이지만 외부 공유 기능 없음.

#### 2-2. 기획 사원 (A2) — 사용자 경험

**제안 1: 캐릭터 커스터마이징 에셋 생성 (우선순위: 높음)**
- **목적**: 이모지 대신 커스텀 캐릭터로 게임 재미 강화
- **필요 기능**:
  - 캐릭터 스프라이트 생성 (2D 픽셀 아트 or 벡터)
  - 표정/동작 바리에이션 (승리, 패배, 대기)
  - 스프라이트 시트 자동 생성
- **사용자 가치**: 캐릭터 애착 ↑, 게임 만족도 ↑
- **시나리오**: 경마 게임에서 말 대신 커스텀 캐릭터가 달리는 모습

**제안 2: UI 아이콘 세트 생성 MCP (우선순위: 높음)**
- **목적**: 텍스트 버튼 대신 직관적 아이콘 버튼 제공
- **필요 기능**:
  - 공통 UI 아이콘 (설정, 볼륨, 준비, 시작, 퇴장 등)
  - SVG 포맷 (크기 무제한 확대, 색상 변경 가능)
  - 다크/라이트 모드 자동 대응
- **사용자 가치**: 조작 직관성 ↑, 모바일 편의성 ↑
- **시나리오**: "준비" 텍스트 대신 체크마크 아이콘, "볼륨" 텍스트 대신 스피커 아이콘

**제안 3: 로딩/전환 애니메이션 생성 (우선순위: 중간)**
- **목적**: 게임 시작/종료 시 시각적 피드백 강화
- **필요 기능**:
  - Lottie JSON 생성 (주사위 굴리는 애니메이션, 룰렛 도는 애니메이션)
  - CSS 기반 로딩 스피너 생성
- **사용자 가치**: 로딩 체감 시간 ↓, 게임 품질 체감 ↑

### 개발팀

#### 2-3. 개발 팀장 (B1) — 아키텍처

**제안 1: SVG 최적화 + 압축 파이프라인 MCP (우선순위: 높음)**
- **목적**: 디자인 리소스 추가 시 성능 저하 방지
- **필요 기능**:
  - SVG 최적화 (SVGO 자동화)
  - PNG/JPG → WebP/AVIF 자동 변환
  - 리소스 캐싱 전략 (브라우저 캐시 + CDN 준비)
- **기술 부채**: 현재 assets/ 폴더에 최적화 없이 원본 파일 사용 중
- **확장성**: MCP로 자동화하면 디자이너가 직접 최적화된 파일 생성 가능

**제안 2: 디자인 토큰 생성 MCP (우선순위: 중간)**
- **목적**: theme.css 변수를 JSON으로 관리, 플랫폼 간 동기화
- **필요 기능**:
  - Figma 디자인 토큰 → CSS 변수 자동 변환
  - JSON → CSS / SCSS / Tailwind 변환
  - 다크/라이트 모드 토큰 자동 생성
- **기술 부채**: 현재 theme.css 수동 관리, 색상 변경 시 전체 파일 수정 필요
- **확장성**: 향후 모바일 앱 (React Native) 진출 시 디자인 토큰 재사용 가능

**제안 3: 스프라이트 시트 생성 MCP (우선순위: 낮음)**
- **목적**: 여러 캐릭터 이미지를 하나의 스프라이트 시트로 통합
- **필요 기능**:
  - 개별 PNG → 스프라이트 시트 자동 생성
  - CSS 애니메이션 코드 자동 생성
  - 프레임 단위 제어 (FPS 조절)
- **기술 부채**: 현재 이모지 기반이라 스프라이트 불필요, 향후 커스텀 캐릭터 도입 시 필요

#### 2-4. 개발 사원 (B2) — 구현

**제안 1: Favicon Generator MCP (우선순위: 높음)**
- **목적**: 브랜드 로고 → 멀티 해상도 파비콘 자동 생성
- **필요 기능**:
  - 입력: SVG/PNG 로고
  - 출력: favicon.ico (16x16, 32x32, 48x48) + apple-touch-icon.png (180x180) + manifest 아이콘 (192x192, 512x512)
  - 자동 HTML 태그 생성
- **영향 파일**: 모든 HTML 파일 `<head>` 섹션
- **구현 난이도**: 낮음 (sharp 라이브러리 기반 MCP 구현 가능)

**제안 2: Image Placeholder Generator MCP (우선순위: 중간)**
- **목적**: 이미지 로딩 전 블러 플레이스홀더 자동 생성
- **필요 기능**:
  - 입력: 원본 이미지
  - 출력: Base64 인코딩된 10px 블러 이미지 (LQIP - Low Quality Image Placeholder)
  - HTML `<img>` 태그 자동 생성 (loading="lazy" + LQIP)
- **영향 파일**: horse-race-multiplayer.html (배경 이미지 로딩)
- **구현 난이도**: 낮음 (plaiceholder 라이브러리 기반)

**제안 3: CSS Gradient Generator MCP (우선순위: 낮음)**
- **목적**: 프롬프트 입력 → CSS 그라디언트 자동 생성
- **필요 기능**:
  - 텍스트 프롬프트 ("sunrise gradient", "ocean waves") → CSS linear-gradient
  - theme.css 변수 자동 추가
- **영향 파일**: css/theme.css (게임별 그라디언트 섹션)
- **구현 난이도**: 중간 (AI 모델 연동 필요)

### 디자인팀

#### 2-5. UI/UX 디자이너 (C)

**제안 1: Icon Library Generator MCP (우선순위: 높음)**
- **목적**: 프로젝트 전용 SVG 아이콘 라이브러리 생성
- **필요 기능**:
  - Heroicons/Lucide 스타일 일관성 유지
  - 커스텀 아이콘 추가 (주사위, 경마, 룰렛 테마)
  - SVG Sprite 자동 생성 (symbol 방식)
  - React/HTML 컴포넌트 자동 생성
- **UX 개선**: 텍스트 버튼 → 아이콘 버튼 (모바일 터치 영역 확대)
- **모바일 호환성**: 작은 화면에서 아이콘이 텍스트보다 공간 효율적

**제안 2: Color Palette Generator MCP (우선순위: 중간)**
- **목적**: 접근성 기준 준수 색상 팔레트 자동 생성
- **필요 기능**:
  - WCAG 2.1 AA 기준 명암비 검증 (4.5:1 이상)
  - 기본 색상 입력 → 50~900 단계 자동 생성
  - theme.css 변수 자동 추가
  - 다크 모드 색상 자동 변환
- **UX 개선**: 색맹 사용자 고려, 접근성 ↑
- **모바일 호환성**: 야외 사용 시 고명암비 색상 가독성 ↑

**제안 3: Responsive Image Resizer MCP (우선순위: 낮음)**
- **목적**: 배경 이미지 → 다중 해상도 자동 생성
- **필요 기능**:
  - 입력: 원본 PNG
  - 출력: @1x, @2x, @3x (Retina 대응)
  - srcset 자동 생성
- **모바일 호환성**: 저해상도 기기에서 로딩 시간 ↓

#### 2-6. 사운드 디자이너 (D)

**제안 1: Sound Effect Generator MCP (우선순위: 높음)**
- **목적**: 프롬프트 기반 효과음 자동 생성
- **필요 기능**:
  - 텍스트 프롬프트 ("dice roll sound", "victory fanfare") → MP3/WAV 생성
  - AI 기반 사운드 생성 (ElevenLabs Audio API 등)
  - 길이/볼륨 자동 조절
- **몰입감**: 현재 무료 효과음 사용 중, 커스텀 효과음으로 브랜드 정체성 ↑
- **적용 시점**: 게임 시작, 승리, 패배, 버튼 클릭

**제안 2: BGM Loop Generator MCP (우선순위: 중간)**
- **목적**: 끊김 없는 배경 음악 루프 생성
- **필요 기능**:
  - 입력: 원본 BGM
  - 출력: 완벽한 루프 포인트 탐지 + 자동 편집
  - 페이드 인/아웃 자동 추가
- **몰입감**: 현재 BGM이 루프 시 끊김 현상 발생 가능
- **적용 시점**: 모든 게임 대기 화면

**제안 3: Audio Mixer MCP (우선순위: 낮음)**
- **목적**: 여러 사운드 레이어 자동 믹싱
- **필요 기능**:
  - BGM + 효과음 + 음성 → 볼륨 밸런스 자동 조절
  - 다이나믹 레인지 압축
- **몰입감**: 게임 중 소리가 너무 크거나 작은 문제 해결

### 품질팀

#### 2-7. QA 엔지니어 (E) — 테스트 방법론

**제안 1: Visual Regression Testing MCP (우선순위: 높음)**
- **목적**: 디자인 변경 시 의도하지 않은 UI 깨짐 자동 탐지
- **테스트 방법**:
  - Percy, Chromatic 스타일 스크린샷 비교
  - 게임 화면 캡처 → 이전 버전과 픽셀 단위 비교
  - 차이 발견 시 자동 리포트 생성
- **자동화 가능**: 높음 (Playwright + pixelmatch 라이브러리)
- **검증 기준**: 99% 픽셀 일치율 (1% 여유는 안티앨리어싱 차이)

**제안 2: Accessibility Audit MCP (우선순위: 중간)**
- **목적**: 색상 명암비, 키보드 접근성 자동 검증
- **테스트 방법**:
  - WCAG 2.1 AA 기준 자동 검증
  - 색상 조합별 명암비 계산
  - 키보드 네비게이션 경로 자동 확인
- **자동화 가능**: 높음 (axe-core 라이브러리)
- **검증 기준**: WCAG AA 모든 항목 통과

**제안 3: Performance Budget MCP (우선순위: 중간)**
- **목적**: 디자인 리소스 추가 시 성능 저하 방지
- **테스트 방법**:
  - 페이지 로딩 시간 자동 측정 (Lighthouse CI)
  - 이미지 총 용량 < 2MB 제한
  - FCP (First Contentful Paint) < 1.5초 제한
- **자동화 가능**: 높음 (Lighthouse CI + GitHub Actions)
- **검증 기준**: 모바일 Lighthouse 점수 > 90점

### 관점 간 충돌 사항

**1. 캐릭터 커스터마이징 vs. 아이콘 우선순위**
- **A2 (기획 사원)**: 캐릭터 커스터마이징 1순위 (게임 재미 ↑)
- **C (UI/UX)**: 아이콘 라이브러리 1순위 (모든 게임 공통 필요)
- **합의 결과**: 아이콘 우선, 캐릭터는 2단계로 보류

**2. 사운드 생성 vs. 이미지 생성 우선순위**
- **D (사운드)**: 효과음 생성 1순위
- **C (UI/UX)**: 아이콘 생성 1순위
- **합의 결과**: 아이콘 1순위, 사운드 2순위 (현재 무료 효과음으로 충분)

**3. 자동화 vs. 수동 디자인**
- **B1 (개발 팀장)**: MCP로 완전 자동화
- **C (UI/UX)**: 디자이너 수동 작업 + MCP 보조
- **합의 결과**: MCP는 "생성 도구"로만 사용, 최종 디자인은 수동 검수
