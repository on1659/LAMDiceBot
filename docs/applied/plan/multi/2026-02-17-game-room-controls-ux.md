# LAMDiceBot 팀 회의록

**일시**: 2026-02-17
**주제**: 게임 방 내부 UX 통일 - 준비/로그아웃/나가기/방 제목 변경 등 컨트롤 일관성
**참석자**: 기획자, 개발자, QA/UX
**회의 방식**: 3인 멀티에이전트 독립 분석 → 교차 검토 → 합의 도출

---

## 1. 현황 요약

### 현재 상태
**문제**: 게임 방 안에 들어간 후의 컨트롤 UI가 게임마다 다름

**주사위**:
- 준비 버튼: 중앙 하단 (준비 섹션 내)
- 나가기 버튼: 우측 상단
- 로그아웃: 있음 (`logout()` 함수)
- 방 제목 변경: 호스트 전용, 인라인 편집
- 호스트 배지: 있음
- 히스토리 패널: 우측 고정

**경마**:
- 준비 버튼: 우측 (준비 섹션 내)
- 나가기 버튼: 중앙 상단
- 로그아웃: 없음
- 방 제목: 고정 ("경마 방")
- 호스트 배지: 있음 (👑 호스트)
- 히스토리 패널: 없음

**룰렛**:
- 준비 시스템: 없음 (즉시 진행)
- 나가기 버튼: 위치 확인 필요
- 로그아웃: 없음
- 방 제목: 고정
- 호스트 배지: 있음
- 히스토리 패널: 우측 고정

**크레인**:
- 준비 버튼: 우측 (준비 섹션 내)
- 나가기 버튼: 위치 확인 필요
- 로그아웃: 없음
- 호스트 배지: 있음 (👑 호스트)
- 호스트 위임 UI: 있음 (플레이어 액션 다이얼로그)

### 문제점
- 각 게임마다 컨트롤 버튼 위치가 다름
- 로그아웃 기능이 주사위에만 있어 일관되지 않음
- 방 제목 변경 UI가 일관되지 않음 (주사위: 편집 가능, 경마/룰렛: 고정)
- 호스트 권한 표시 방법이 다름
- 준비 버튼 위치가 게임마다 다름

## 2. 독립 분석 결과

### 2-1. 기획자 (A)

#### 제안 1: 통합 헤더 네비게이션 바 도입 - 우선순위: 상
- **전략적 의의**: 모든 게임에서 일관된 탐색 경험 제공 → 브랜드 정체성 강화, 학습 곡선 최소화
- **사용자 가치**: 어떤 게임이든 동일 위치에서 "나가기", "설정", "로그아웃" 접근, 여러 게임 번갈아 플레이 시 직관적
- **이유**: 가장 기본적인 개선사항, 사용자가 가장 먼저 체감

#### 제안 2: 로그아웃 기능 추가 (닉네임 변경 지원) - 우선순위: 중
- **전략적 의의**: 현재 브라우저 새로고침 필요 → 즉시 닉네임 변경 가능, PC방/모임 환경에서 유용
- **사용자 가치**: 방 나가지 않고 사용자명 변경, 같은 기기 여러 명 번갈아 사용 시 편리
- **이유**: 필수는 아니지만 실제 사용 시나리오에서 자주 발생하는 불편함 해소

#### 제안 3: 방 제목 편집 UI 통일 및 접근성 개선 - 우선순위: 중
- **전략적 의의**: 주사위는 편집 가능, 경마/룰렛은 고정 → 모든 게임에서 통일하여 커뮤니티 기능 강화
- **사용자 가치**: 호스트가 방 목적 명시 (예: "친구들끼리만", "초보 환영"), 방 찾기 용이
- **이유**: 소셜 기능 강화, 멀티플레이어 플랫폼 정체성

#### 제안 4: 준비 시스템 표준화 (주사위 게임 포함) - 우선순위: 하
- **전략적 의의**: 현재 주사위는 즉시 시작, 경마/룰렛/크레인은 준비 시스템 → 흐름 차이로 혼란
- **사용자 가치**: 게임별 룰을 빠르게 이해, 초보 사용자 진입 장벽 낮춤
- **이유**: 게임 특성상 다른 시작 방식이 합리적일 수 있음, 시각적 가이드로 차이 명시

#### 제안 5: 호스트 권한 표시 및 위임 UI 일관성 - 우선순위: 상
- **전략적 의의**: 호스트 배지는 있지만 위임 방식이 다름 → 통일하여 방 관리 신뢰성 향상
- **사용자 가치**: 호스트 명확히 인식, 호스트 교체 시 혼란 없음
- **이유**: 방 관리는 멀티플레이어 핵심, 오류 시 사용자 불만 직결

---

### 2-2. 개발자 (B)

#### 제안 1: 공통 컨트롤 바 컴포넌트화 - 난이도: 중
- **구현 방법**: 모든 게임 공통 "room-controls-bar" 상단 배치 (방 제목/사용자명/호스트 뱃지/준비/로그아웃/나가기)
- **영향 파일**: 모든 게임 HTML 4개 + `css/theme.css`
- **이유**: 각 게임마다 현재 구조가 다르지만, 로직 변경은 최소화 (기존 함수 재사용)

#### 제안 2: 방 내 로그아웃 기능 추가 - 난이도: 중
- **구현 방법**: 주사위의 `logout()` 함수를 모든 게임에 추가, Socket 이벤트 `changeUserName` 추가
- **영향 파일**: 경마/룰렛/크레인 HTML + `socket/shared.js` 또는 각 게임별 핸들러 + `socket/index.js`
- **이유**: 서버 측 로직 추가 필요 (사용자명 중복 검증, 방 내 알림)

#### 제안 3: 방 제목 변경 UI 통일 - 난이도: 하
- **구현 방법**: 주사위 패턴을 경마/룰렛/크레인에 적용, 방 제목 옆 ✏️ 아이콘 → 인라인 편집
- **영향 파일**: 경마/룰렛/크레인 HTML (Socket 이벤트 `updateRoomName`은 이미 존재)
- **이유**: 프론트엔드 HTML/CSS만 수정, 서버 로직 이미 존재

#### 제안 4: 준비 버튼 위치 통일 (우측 상단 컨트롤 바 내) - 난이도: 하
- **구현 방법**: 모든 게임에서 준비 버튼을 상단 컨트롤 바 우측에 배치 (나가기 버튼 옆)
- **영향 파일**: 모든 게임 HTML 4개 + `css/theme.css`
- **이유**: HTML 구조 변경만 필요, JavaScript 로직 재사용

#### 제안 5: 통합 게임 컨트롤 모듈 (room-controls-shared.js) - 난이도: 상
- **구현 방법**: 방 제목 변경/준비/로그아웃/나가기 등 공통 기능을 독립 JS 모듈로 분리
- **영향 파일**: `public/js/room-controls-shared.js` (신규) + 모든 게임 HTML 4개
- **이유**: 새 모듈 아키텍처 설계 필요, 게임별 특수 로직 처리 필요

### 2-3. QA/UX (C)

#### 제안 1: 상단 고정 컨트롤 바 - 리스크: 하
- **테스트 방법**: A/B 테스트, 시선 추적, 태스크 완료 테스트, 크로스 게임 일관성, 모바일 터치 영역
- **UX 개선점**: 학습 비용 제거, 예측 가능성, 오류 방지, 정보 계층 구조 명확화
- **이유**: 레이아웃 변경 단순, CSS 변수 시스템 활용 용이

#### 제안 2: 준비 버튼 위치 표준화 (좌측 패널 하단 고정) - 리스크: 중
- **테스트 방법**: 클릭 히트맵, 태스크 시간 측정, 오클릭률, 멀티게임 세션, 눈 깜빡임 비율
- **UX 개선점**: 인지 부하 감소, 행동 유도성, 모바일 최적화, 상태 피드백
- **이유**: 경마 탈것 선택 섹션과 우선순위 충돌, 주사위 드래그 앤 드롭 준비 기능 충돌 가능성

#### 제안 3: 방 제목 변경 UI 통일 (인라인 편집) - 리스크: 하
- **테스트 방법**: 사용성 테스트, 에러 발생률, 클릭 수 측정, 인지적 워크쓰루, A/B 테스트
- **UX 개선점**: 직접 조작, 피드백 즉시성, 시각적 일관성, 접근성 (키보드 네비게이션)
- **이유**: 주사위 이미 인라인 편집, 경마는 서버 로직 변경 필요

#### 제안 4: 호스트 권한 시각화 통일 - 리스크: 하
- **테스트 방법**: 시각적 스캔, 색맹 사용자, 멀티모달, 사용자 인터뷰, 접근성 감사
- **UX 개선점**: 보편적 상징성 (👑), 중복 인코딩, 계층 강조, 브랜드 일관성
- **이유**: 이미 CSS 변수 `--host-badge-bg` 정의, 주사위/경마 호스트 뱃지 사용 중

#### 제안 5: 로그아웃 기능 전역 추가 + 확인 다이얼로그 - 리스크: 중
- **테스트 방법**: 오류 복구 시나리오, 사용자 실수 로그, 다이얼로그 문구 A/B, 태스크 완료율, 세션 복원
- **UX 개선점**: 기능 완전성, 실수 방지, 컨텍스트 보존, 일관된 용어 ("나가기" vs "로그아웃")
- **이유**: 로그아웃 로직 통일 필요, 세션 복원 로직(sessionStorage) 게임마다 다름

## 3. 교차 검토 결과

### 3-1. 기획자 검토

#### 개발자(B) 제안 평가
- **B-1 공통 컨트롤 바**: **긍정** - 저비용 고효율, 유지보수 효율, 브랜드 일관성
- **B-2 로그아웃 기능**: **긍정** - Pain Point 해결, 보안 강화, CS 비용 감소
- **B-3 방 제목 통일**: **긍정** - 낮은 비용, 즉각적 체감, ROI 매우 높음
- **B-4 준비 버튼**: **긍정** - 고빈도 기능, 모바일 최적화, 실수 방지
- **B-5 통합 모듈**: **중립** - 장기 확장성 vs 단기 투자 부담, 게임 6개 이상 시점 재검토

#### QA/UX(C) 제안 평가
- **C-1 상단 고정 바**: **긍정** - 모던 웹 앱 표준, 스크롤 편의성, 모바일 대응
- **C-2 준비 버튼 표준화**: **긍정** - 직관성 향상, A/B 테스트 권장, 툴팁으로 완화
- **C-3 방 제목 인라인 편집**: **긍정** - 편의성, 플랫폼 표준 패턴, 호스트 권한 강화
- **C-4 호스트 권한 시각화**: **긍정** - 명확한 역할 인식, 커뮤니티 관리, 향후 확장 기반
- **C-5 로그아웃 + 다이얼로그**: **긍정** - PC방 시나리오 필수, 실수 방지, 신뢰도 향상

### 3-2. 개발자 검토

#### 기획자(A) 제안 평가

- **A-2 로그아웃**: **조건부 가능** - 사양 명확화 필요 (방 나가기 vs 세션 초기화)
- **A-3 방 제목 편집**: **가능** - 패턴 이미 존재, 적용 범위 확대만
- **A-4 준비 시스템**: **가능** - 이미 모든 게임에 준비 섹션, 표시 조건 통일만
- **A-5 호스트 권한**: **가능** - 서버 로직 완성, 클라이언트 UI만 통일

#### QA/UX(C) 제안 평가
- **C-1 상단 고정 바**: **가능** - CSS 변경 위주, 레이아웃 충돌 주의
- **C-2 준비 버튼 표준화**: **조건부 가능** - 현재 "좌측 패널" 없음, 레이아웃 재구성 필요
- **C-3 방 제목 인라인**: **가능** - A-3과 동일
- **C-4 호스트 시각화**: **가능** - A-5와 동일
- **C-5 로그아웃 + 다이얼로그**: **조건부 가능** - A-2와 동일

- **B-5 통합 모듈**: **긍정 (장기)** - 유지보수 효율, 과도한 추상화 주의, 충분한 QA

## 6. 회의록 요약

### 핵심 결정
1. **방향성**: "더 나은 쪽으로 통일" - 주사위의 로그아웃/방 제목 편집 기능을 다른 게임에 확장
2. **단계별 접근**: 낮은 난이도 항목(방 제목, 호스트 권한, 준비 버튼)부터 구현 → 공통 컨트롤 바 → 로그아웃
3. **사양 명확화 필요**: 로그아웃 = "방 나가기 + 메인 페이지" vs "닉네임 재설정" 중 선택

### 기대 효과
- 신규 사용자 학습 곡선 제거 (게임 전환 시 재학습 불필요)
- 유지보수 효율성 향상 (한 번 수정으로 모든 게임 반영)
- 플랫폼 브랜드 일관성 강화
- PC방 사용자 니즈 충족 (로그아웃 기능)
