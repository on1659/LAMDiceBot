# Cursor IDE 메모리 최적화 가이드

## 문제
Cursor IDE가 메모리 부족으로 인해 느려지거나 오류가 발생하는 경우

## 해결 방법

### 1. Cursor 설정 최적화

설정 파일 위치: `%APPDATA%\Cursor\User\settings.json`

추가된 메모리 최적화 설정:
- **파일 감시 제외**: node_modules, dist, build 등 대용량 폴더 제외
- **검색 범위 제한**: 불필요한 폴더 검색에서 제외
- **TypeScript 서버 메모리 제한**: 4GB로 제한
- **자동 완성 최적화**: package.json 자동 import 비활성화
- **시각적 기능 비활성화**: 코드 렌즈, 미니맵, 시맨틱 하이라이팅 비활성화
- **확장 프로그램 자동 업데이트 비활성화**

### 2. MCP 서버 설정 (선택사항)

MCP (Model Context Protocol) 서버를 사용하여 일부 기능을 외부 프로세스로 분리할 수 있습니다.

설정 파일 위치: `%APPDATA%\Cursor\User\globalStorage\mcp_settings.json` (필요시 생성)

### 3. 추가 최적화 팁

1. **불필요한 확장 프로그램 비활성화**
   - 사용하지 않는 확장 프로그램은 비활성화하거나 제거

2. **대용량 파일 처리**
   - 큰 파일은 별도 에디터에서 열기
   - 파일 크기 제한 설정 활용

3. **Git 기능 최적화**
   - 큰 저장소의 경우 Git 기능 일시 비활성화

4. **워크스페이스 정리**
   - 불필요한 파일 삭제
   - .gitignore에 대용량 파일 추가

## 적용 방법

1. Cursor IDE 재시작
2. 설정이 자동으로 적용됨
3. 메모리 사용량 모니터링

## 확인 방법

- 작업 관리자에서 Cursor 프로세스 메모리 사용량 확인
- Cursor 내부 성능 모니터 도구 사용
