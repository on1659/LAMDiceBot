## 상세 변경 내역

### 워크플로우 실행 그래프 신규 생성
- 개발 파이프라인 상태 전이표: TRIAGE → SCOUT → SPEC → CODE ←→ REVIEW ←→ QA → DONE
- 계획 파이프라인 상태 전이표: CONTEXT → CLAUDE_DRAFT → DEBATE_LOOP → SYNTHESIS → DONE
- 분기 조건 명시: Reviewer approve/request-changes, QA pass/fail
- 루프 정책: Reviewer→Coder max 3, QA→Coder max 3, 2회 교착 시 이더 개입
- ESCALATE 상태: 루프 초과 시 미해결 이슈 + 결과를 사용자에게 보고
- 재트리아지 전이: 조사→수정 전환, SIMPLE 영향 확대, Scout 보고 초과
- 트랙 간 연결: impl 문서가 계획→개발 인터페이스 역할

### 하네스 재트리아지 규칙
- "확인"과 "수정"은 별개 단계 — 자연 전환 금지
- 조사 중 수정 필요 → 트리아지 재수행
- Scout 영향 범위가 최초 수준 초과 → 상향 재판정

### Coder 실행 절차 추가
- 구현 전(1-3): 수정 파일 읽기, 호출부 grep, 유사 구현 참조
- 구현 중(4-5): 요청 외 리팩터링 금지, 불변조건 재확인
- 구현 후(6-10): 시그니처 grep, node -c 문법, 영향 보고, QA 체크리스트, 테스트 불가 사유
