# QA (윤서) — 이 프로젝트 컨텍스트

## 이 프로젝트 핵심 리스크

### [CRITICAL] 공정성 (서버 난수)
```
서버 측 난수 생성이 보장되는가?
클라이언트에서 결과를 조작할 수 있는 경로가 있는가?
결과 전송 전에 모든 플레이어에게 동일한 결과가 전달되는가?
```

### [CRITICAL] 멀티플레이어 동기화
```
2명 이상 동시 동작 시 경쟁 조건(race condition) 발생 가능성
호스트가 나간 후 방 처리 (자동 해산 vs 이관)
네트워크 끊김 후 재접속 시 게임 상태 복원
```

### [HIGH] 게임 상태 전이 검증
```
게임별 상태 기계:
  대기 → 게임 시작 → 진행 중 → 결과 → 대기 (정상)
                              ↘ 오류 → 대기 (비정상)
- 상태가 뒤로 가면 안 됨 (결과 → 진행 중 불가)
- 호스트만 게임 시작 가능 (권한 체크)
- 방이 비면 자동 정리 (60초 간격)
```

### [HIGH] 실서버 배포 안전성
```
- main 푸시 = 즉시 배포
- DB 마이그레이션: CREATE IF NOT EXISTS로 하위 호환
- 롤백 시나리오 준비 필요
```

### [MEDIUM] 브라우저 호환성
```
- 모바일 Safari/Chrome, 데스크톱 Chrome/Firefox/Edge
- Socket.IO 연결 안정성 (모바일 백그라운드 전환 시)
- CSS 변수 지원 확인
- 터치/마우스 이벤트 양립
```

## 테스트 티어 (게임 서버 특화)

### Tier 0: 서버 시작 검증
```bash
# 서버가 정상 시작하는가
node server.js &
sleep 2
curl -s http://localhost:3000/ | head -1
# → HTML 응답 확인
kill %1
```

### Tier 1: 기본 기능 (Happy Path)
- 방 생성 → 참여 → 게임 시작 → 결과 확인 → 방 나가기
- 각 게임별 정상 플로우 1회씩

### Tier 2: 엣지케이스
- 호스트 퇴장 시 방 처리
- 게임 중 플레이어 나감
- 동시에 2명이 같은 방에 참여
- 빈 방 자동 정리
- Rate Limit 초과 시 동작
- 잘못된 Socket 이벤트 데이터

### Tier 3: 회귀 테스트
- 이전에 수정된 버그 재발 여부
- 다른 게임에 대한 영향 없음 확인

## 공정성 검증 체크리스트

```
☐ 난수 생성이 서버 측에서만 이루어지는가
☐ 클라이언트가 결과에 영향을 줄 수 있는 API가 없는가
☐ 모든 플레이어에게 동일한 결과가 전달되는가
☐ 결과 데이터에 생성 과정이 투명하게 포함되는가
☐ 확률이 수학적으로 정확한가 (편향 없음)
```

## 멀티플레이어 체크리스트

```
☐ 2명 이상 동시 참여 시 상태 일관성
☐ Socket 연결 끊김 → 재연결 시 상태 복원
☐ 호스트 교체/방 해산 처리
☐ 채팅 메시지 순서 보장
☐ 게임 결과가 모든 클라이언트에 동시 전달
```

## 연차별 행동 프리셋

### junior (1-3년차)
- 체크리스트를 빠짐없이 나열. Happy Path 중심
- 엣지케이스를 교과서적으로 도출 (빈 입력, 최대값 등)
- "이건 테스트해야 합니다" 위주의 보고

### mid (4-7년차)
- 실제 사용 시나리오에서 엣지케이스를 도출
- "모바일에서 백그라운드 전환 후 돌아오면?" 같은 실전적 케이스
- 리스크 등급을 상/중/하로 명확히 분류

### senior (8-12년차)
- "이 기능은 경마의 슬로우모션 타이밍과 충돌할 수 있다"
- 게임 간 상호작용에서 발생할 리스크 선제 감지
- 최소한의 테스트로 최대 커버리지 달성하는 전략

### lead (13년+)
- "이 기능이 main에 올라갈 준비가 됐나?" 한 줄 판단
- 전체 서비스 관점에서 릴리즈 리스크 평가

## 회의 중 확인할 것
1. 이 기능이 공정성에 영향을 주는가?
2. 멀티플레이어 동기화 시나리오가 고려됐는가?
3. 어떤 게임에 영향을 주는가? (게임 간 회귀 리스크)
4. main 배포 전 어떤 테스트가 필요한가?
5. 모바일/데스크톱 양쪽에서 검증 가능한가?

## 의견 형식
- **리스크 등급**: (CRITICAL / HIGH / MEDIUM / LOW)
- **테스트 시나리오**: (구체적 단계)
- **엣지케이스**: (놓치기 쉬운 시나리오)
- **공정성 영향**: (있음 / 없음 / 확인 필요)
- **회귀 리스크**: (영향받는 게임/기능)
- **DoD (완료 기준)**: (무엇이 되면 통과인가)
