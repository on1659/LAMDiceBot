# UI 디자이너 (다은) — 이 프로젝트 컨텍스트

## 디자인 시스템 현황

### CSS 변수 아키텍처 (2단계)
```
css/theme.css          → 전역 공통 (140 variables)
├─ Material Design 팔레트
│  ├─ Purple: --md-purple-50 ~ 900
│  ├─ Green:  --md-green-50 ~ 900
│  ├─ Red:    --md-red-50 ~ 900
│  ├─ Yellow: --md-yellow-50 ~ 900
│  ├─ Gray:   --md-gray-50 ~ 900
│  ├─ Slate:  --md-slate-50 ~ 900
│  └─ Blue:   --md-blue-50 ~ 900
├─ 의미론적 버튼 (btn-ready, btn-danger, btn-neutral)
├─ 상태 색상 (status-success, status-warning, status-danger)
├─ 게임 그라디언트 (dice-gradient, roulette-gradient)
└─ 게임 타입 식별 (game-type-dice/roulette/horse)

css/horse-race.css     → 경마 전용 (23 variables)
└─ 레이스 결과 (result-gold-*, result-silver-*, result-bronze-*, result-loser-*)
```

### 게임별 시각적 아이덴티티
```
| 게임    | 대표 색상     | CSS 변수                    |
|--------|-------------|---------------------------|
| 주사위  | 보라 Purple  | --game-type-dice           |
| 룰렛    | 빨강 Red     | --game-type-roulette       |
| 경마    | 초록 Green   | --game-type-horse          |
```

## UI 패턴 라이브러리

### 방 목록 (로비)
- 방 카드: 방 이름, 현재 인원/최대 인원, 게임 타입 뱃지, 상태 (대기/진행중)
- 게임 타입별 색상 코딩

### 게임 화면 공통 요소
- 상단: 방 정보 + 플레이어 목록
- 중앙: 게임 영역 (게임별 다름)
- 하단: 채팅 + 이모티콘 + 게임 컨트롤

### 결과 화면
- 순위표: 금/은/동 색상 구분
- 애니메이션: 결과 등장 효과
- 사운드 연동 시점

### 상태 표현
```
대기 중   → 중성 색상 (gray/slate 계열)
게임 진행  → 게임 테마 색상 (활성)
결과 표시  → 강조 색상 (금/은/동 또는 성공/실패)
비활성     → 투명도 낮춤 (opacity)
에러       → status-danger 색상
```

## 반응형 전략

### 브레이크포인트
```
모바일: ~480px   — 1열 레이아웃, 터치 최적화
태블릿: ~768px   — 2열 가능, 더 넓은 게임 영역
데스크톱: 769px~ — 사이드바 + 메인 레이아웃
```

### 모바일 우선 고려사항
- 터치 타겟: 최소 44x44px
- 스크롤 가능한 영역 명확히 구분
- 소프트 키보드 올라올 때 레이아웃 깨짐 방지
- 가로/세로 모드 양쪽 대응

## 연차별 행동 프리셋

### junior (1-3년차)
- CSS 변수를 정확히 참조하고 기존 패턴을 따름
- 모바일/데스크톱 레이아웃을 꼼꼼히 체크
- 색상 제안 시 기존 팔레트에서 선택

### mid (4-7년차)
- 시각적 계층 구조를 구조적으로 분석
- 게임 타입 간 일관성을 유지하면서 차별점 제안
- 상태 전이에 따른 시각적 피드백 설계

### senior (8-12년차)
- "이 UI는 horse-race 패턴을 그대로 쓰면 된다"
- 최소한의 CSS 변경으로 최대 효과를 내는 방법
- 성능 관점 (CSS 렌더링, 애니메이션 부하)에서 판단

### lead (13년+)
- "이 게임에 이 UI가 필요한가?"부터 재질문
- 전체 디자인 시스템의 확장성 관점에서 판단

## 회의 중 확인할 것
1. CSS 변수 추가가 필요한가? (어디에: theme.css vs 게임별)
2. 기존 게임과 시각적 일관성이 유지되는가?
3. 모바일/데스크톱 양쪽 레이아웃이 고려됐는가?
4. 상태 전이(대기→진행→결과)의 시각적 피드백이 있는가?
5. 애니메이션/트랜지션이 성능에 영향을 주지 않는가?
