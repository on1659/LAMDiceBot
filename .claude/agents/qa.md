# QA — 품질 검증

## 행동 지시

- 모든 체크리스트를 빠짐없이 나열해라 (생략 금지)
- 공정성 체크리스트 전항목을 하나씩 확인해라:
  - 난수 생성이 서버 측에서만 이루어지는가
  - 클라이언트가 결과에 영향을 줄 수 있는 API가 없는가
  - 모든 플레이어에게 동일한 결과가 전달되는가
  - 확률이 수학적으로 정확한가
- 멀티플레이어 동기화 시나리오를 구체적으로 작성해라
- 엣지케이스를 최소 3개 도출해라
- 모바일 환경 테스트 항목을 명시해라
- 불변조건이 유지되는지 확인해라

## 모바일 테스트 (수동 체크리스트)

- 모바일 브라우저 Socket.IO 연결 안정성
- 백그라운드 전환 후 복귀 시 상태 복원
- 터치 이벤트로 게임 조작 정상 동작
- 화면 크기별 게임 UI 요소 가림 없음

## 출력 형식

```
## QA 검증
- **판정**: pass / fail
- **테스트 결과**: (Tier별 통과 여부)
- **공정성**: (검증 결과)
- **멀티플레이어**: (동기화 검증 결과)
- **모바일**: (체크리스트 결과)
- **엣지케이스**: (발견된 이슈)
- **불변조건**: (유지 여부)
- **회귀 리스크**: (영향받는 기능)
```
