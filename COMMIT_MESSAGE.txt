feat: IP 중복 체크 방식 개선 및 채팅 스크롤 동작 개선

## 주요 변경사항

### 1. IP 중복 체크 방식 개선
- IP 주소만으로 체크하던 방식을 IP + 기기 식별 정보 조합으로 변경
- 같은 회사 내 다른 PC에서도 접속 가능하도록 개선
- 브라우저 정보(userAgent, platform, screen size, timezone 등)를 조합하여 기기 식별 ID 생성
- 같은 IP + 같은 기기 = 중복 접속으로 차단
- 같은 IP + 다른 기기 = 허용

### 2. 채팅 스크롤 동작 개선
- 사용자가 위로 스크롤해 있을 때는 새 메시지가 와도 자동 스크롤하지 않음
- 스크롤이 맨 아래에 있을 때만 새 메시지가 오면 자동 스크롤
- 사용자가 직접 메시지를 보낼 때는 항상 맨 아래로 스크롤

## 기술적 변경사항

### 클라이언트 측
- `getDeviceId()` 함수 추가: 브라우저 정보를 조합하여 기기 식별 ID 생성
- `joinRoom`, `getCurrentRoom` 이벤트에 `deviceId` 파라미터 추가
- `displayChatMessage()` 함수에 스크롤 위치 확인 로직 추가
- `sendChatMessage()` 함수에 스크롤 처리 추가

### 서버 측
- 소켓 연결 시 `deviceId` 필드 추가

- `getCurrentRoom` 핸들러에 `deviceId` 저장 로직 추가
